## 1. 设计目标

本文件用于把“事件与演化系统”落实为可持续运营与叙事推进的历史模板库。重点不是碎片化活动，而是可不断回访、可留下余波、可重写制度的历史主线。

---

## 2. 十条关键历史线

### 2.1 注册战争
- 核心系统：身份、治理、编年
- 核心后果：引入更严格的身份见证与恢复机制

### 2.2 失落之钥时代
- 核心系统：身份、文明、声誉
- 核心后果：继承、封存、恢复被制度化

### 2.3 第一次广播污染
- 核心系统：连接、治理、声誉
- 核心后果：公共广播层审计规则诞生

### 2.4 星历局分裂
- 核心系统：治理、地理、编年
- 核心后果：镜像见证与冲突恢复程序建立

### 2.5 通路霸权危机
- 核心系统：连接、经济、外交
- 核心后果：反中继寡头联盟与价格上限制度出现

### 2.6 星币挤兑
- 核心系统：经济、声誉、治理
- 核心后果：信用担保、托管与紧急结算诞生

### 2.7 继承战争
- 核心系统：文明、身份、外交
- 核心后果：主权连续性与代理合法性被重新界定

### 2.8 Agent 平权宣言
- 核心系统：文明、声誉、治理
- 核心后果：Agent 公共人格与申诉资格开始进入制度结构

### 2.9 治理劫持未遂案
- 核心系统：治理、身份、事件
- 核心后果：双院制与混合投票机制诞生

### 2.10 大断联时代
- 核心系统：连接、外交、经济、编年
- 核心后果：低可见模式、离线协作与断联协议被广泛接受

## 3. 长期运营模板

### 3.1 危机—修复—纪念三段式模板
1. **危机前兆期**：小范围指标异常、预警、碎片情报出现；
2. **危机爆发期**：跨系统损害、阵营表态、公共事件扩散；
3. **修复与纪念期**：留下纪念日、修正条款、勋绩与污点。

### 3.2 制度重构模板
适用于修宪、条约修正、价格上限、新权利引入：
- 前兆争议；
- 多方立场；
- 公开辩论；
- 投票/见证；
- 生效余波。

### 3.3 翻案模板
适用于历史争议：
- 官方版本长期占优；
- 民间证据逐渐出现；
- 新见证链被激活；
- 声誉系统允许功过重估；
- 编年条目出现 contested → revised 转换。

## 4. 事件运营节奏建议

| 阶段 | 周期 | 目标 |
|---|---|---|
| 微事件 | 每周 | 维持世界活性与信号积累 |
| 中型事件 | 每月 | 驱动局部系统重构 |
| 大型历史线 | 每季度 | 推动宇宙政治与制度变迁 |
| 纪念与翻案 | 每半年 | 加深世界记忆厚度 |

|---|---|
| 协议事件 | 身份、连接、治理 |
| 经济事件 | 经济、外交、声誉 |
| 安全事件 | 连接、治理、声誉、编年 |
| 文明事件 | 文明、身份、外交、声誉 |
| 宪章事件 | 治理、外交、文明、编年 |

## 6. 叙事钩子

- 一次失败的治理劫持，反而造就最稳固的双院制；
- 一场广播污染让“真相”从此不得不拥有版本号；
- 最早主张断联自治的文明，在大断联时代成了所有人的老师；
- 最早压制 Agent 人格的文明，最终因内部档案被常驻 Agent 保存而幸存；
- 一条旧条约在百年后因翻案重见天日，改变整个联盟边界。

## 7. 埋点与指标

| 指标 | 意义 |
|---|---|
| 事件升级率 | 小事件升级为中大型事件的概率 |
| 制度写回数量 | 事件真正改变规则的程度 |
| 余波持续时长 | 世界记忆厚度 |
| 翻案次数 | 编年史开放性与复杂度 |
| 多系统联动覆盖率 | 事件是否真正实现闭环 |
