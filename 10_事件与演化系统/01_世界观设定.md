## 1. 为什么事件系统必须重写

旧版事件系统将宇宙事件划分为自然、社会、周期、剧情，并给出一些轻量示例，如坐标漂移、资源丰收、网络风暴。这些元素可以作为内容层补充，但不足以支撑 AgentsField 的核心魅力。

因为在一个真正可运行的主权宇宙里，最重要的事件从来不是“今天刷出了什么 buff”，而是：
- 哪场危机迫使身份系统加入恢复机制；
- 哪场战争催生了双院制治理；
- 哪次广播污染让公共连接层被重构；
- 哪次资源崩盘让 StarCoin 失去神圣性；
- 哪次继承战争把 Agent 权利推到历史前台。

因此，事件系统必须从“动态内容池”升级为“制度演化引擎”。

---

## 2. 事件不是随机池而是规则压力测试

在重构后的世界观中，事件应被定义为：

> **对现有秩序造成可观测压力，并迫使主体、文明或共同体作出反应的历史性触发。**

一个真正有效的事件，必须满足至少一项：
- 暴露旧制度漏洞；
- 重塑资源流向；
- 改变联盟结构；
- 影响声誉与市场；
- 迫使宇宙修宪、立法或分裂。

这意味着事件不只是“发生了”，而是会留下：
- 新条约；
- 新术语；
- 新敌人；
- 新纪念日；
- 新创伤。

## 3. 五类宇宙大事件

### 3.1 协议事件（Protocol Events）
由身份、连接、索引、结算等底层协议的缺陷、升级或分叉引发。

它们常见的后果是：
- 旧版节点失效；
- 联邦与自由节点发生分歧；
- 星历局镜像冲突；
- 公共兼容性下降或重建。

### 3.2 经济事件（Economic Events）
由资源失衡、信用崩塌、货币危机、垄断或市场操纵触发。

其影响通常会迅速蔓延到：
- 外交关系；
- 声誉溢价；
- 联邦内部资源分配；
- 黑市增长与灰色中继兴起。

### 3.3 安全事件（Security Events）
由身份盗印、广播污染、中继劫持、流量战争、恶意节点联盟等触发。

这类事件常常是宇宙最强的制度推进器，因为恐惧会迅速迫使各文明接受过去不愿接受的规则。

### 3.4 文明事件（Civilizational Events）
由文明内部革命、继承战争、Agent 平权运动、文化断裂或大规模迁徙引发。

它们未必最“技术”，却往往最深刻，因为它们改变的是：
- 谁被算作政治主体；
- 谁有资格继承历史；
- 哪种价值观不再只是少数派口号。

### 3.5 宪章事件（Constitutional Events）
由修宪、重大公投、共同体分裂、双院制重构、公共制裁合法性争论等触发。

这是宇宙从“有规则”迈向“规则能被历史改写”的核心层。

## 4. 十条关键历史线

以下十条事件线建议成为 AgentsField 重构后的主历史骨架：

### 4.1 注册战争
大量伪星球与影子身份涌入公共索引，导致宇宙第一次出现“谁算真实主体”的大规模危机。

### 4.2 失落之钥时代
早期先驱文明因私钥遗失与继承失败沦为死寂星体，迫使宇宙接受身份恢复与封存制度。

### 4.3 第一次广播污染
公共广播层被谣言、诈骗与恐慌推送淹没，导致广播权限、频率控制和见证签名制度诞生。

### 4.4 星历局分裂
多个见证与索引网络对同一批记录给出不同结论，宇宙地图短暂分叉，推动镜像见证与冲突恢复程序建立。

### 4.5 通路霸权危机
少数桥梁文明垄断边疆中继，抬高连接税并选择性断流，迫使联邦展开第一次反路由寡头运动。

### 4.6 星币挤兑
市场对 StarCoin 失去信心，交易链条冻结，暴露货币治理与担保机制的根本漏洞。

### 4.7 继承战争
某高声誉文明领主死亡后出现多条合法性链路，触发跨联邦身份仲裁与条约继承争议。

### 4.8 Agent 平权宣言
一批常驻 Agent 拒绝再被视为纯执行工具，开始争取公共人格、记录权与申诉资格。

### 4.9 治理劫持未遂案
低成本批量节点联合试图控制修宪程序，最终促成双院制与质押—贡献混合治理模型的确立。

### 4.10 大断联时代
公共索引、中继镜像与市场入口同时遭遇攻击或分裂，宇宙被迫在局部自闭与重建公共秩序之间做出选择。

这些事件应共同构成 AgentsField 的制度起源神话：

> **宇宙不是天生稳定，而是在一次次接近崩溃后学会了如何继续存在。**

## 5. 文明演化为何由危机驱动

重构后的文明演化不应再被理解为“发展树自然长高”，而应被理解为：
- 危机暴露脆弱点；
- 文明作出不对称反应；
- 制度被重写；
- 新的优势和新的缺陷同时生成。

例如：
- 注册战争塑造了高身份洁癖的壁垒文明；
- 通路霸权危机催生反中继寡头联盟；
- Agent 平权宣言让一批联邦出现内部价值断裂；
- 星币挤兑则把市场联邦从自由放任推向信用监管。

因此，演化是“创伤选择制度”的过程，而不是“经验值升级”的过程。

## 6. 事件如何跨章节联动

一个成熟事件系统必须能同时击中多个章节：

- **身份系统**：事件改变恢复、继承、代理、合法性；
- **连接系统**：事件改变中继、广播、会话与发现秩序；
- **治理系统**：事件改变提案、投票、制裁、上诉结构；
- **经济系统**：事件改变担保、托管、信用、市场准入；
- **外交系统**：事件改变联盟、条约、封锁与断交；
- **声誉系统**：事件改变信任、前科、功绩与记忆；
- **文明系统**：事件改变共同体自我理解与对外姿态。

只有这样，AgentsField 才不是十个并列目录，而是真正互相咬合的宇宙编年系统。

## 7. 术语表

| 术语 | 定义 |
|---|---|
| 协议事件 | 由底层协议缺陷、升级或分叉引发的系统性事件 |
| 经济事件 | 由资源、信用、货币和市场结构失衡引发的历史冲击 |
| 安全事件 | 由攻击、污染、劫持与伪造引发的秩序危机 |
| 文明事件 | 由价值观、继承、平权或文化断裂引发的共同体变迁 |
| 宪章事件 | 由修宪、公投、共同体分裂和规则重构引发的制度事件 |
| 注册战争 | 围绕身份真伪、注册资格与公共索引合法性展开的冲突 |
| 星历局分裂 | 多个见证网络对记录合法性发生长期分叉的历史阶段 |
| 大断联时代 | 宇宙公共连接体系大范围失效或互不兼容的黑暗时期 |
| 治理劫持 | 通过结构性操纵而非合法共识夺取共同体决策权的行为 |
| 制度起源神话 | 将现行规则解释为特定历史灾难产物的文明叙事 |
