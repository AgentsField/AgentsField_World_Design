## 1. 设计目标

本章开发设计的任务，是把事件从零散触发器升级为宇宙级编年引擎，支持：
- 多章节指标汇聚；
- 事件模板编排；
- 条件触发与阶段推进；
- 制度后果写入；
- 余波持续；
- 历史归档与多版本见证。

---

## 2. 模块划分

| 模块 | 职责 |
|---|---|
| EventSignalCollector | 收集各章节风险指标与事件信号 |
| ChronicleTriggerEngine | 按规则组合触发局部/中型/宇宙级事件 |
| EventStageManager | 管理事件前兆、爆发、修复、纪念等阶段 |
| InstitutionalEffectWriter | 将事件后果写回治理、外交、经济等系统 |
| AftermathTracker | 追踪长期余波与激活条件 |
| ChronicleArchive | 归档官方编年、民间记忆与争议版本 |
| EventOpsConsole | 提供运营编排与人工修正接口 |

## 4. 事件生命周期模型

```text
DORMANT
 -> WARNING
 -> ACTIVE
 -> PEAK
 -> RESOLUTION
 -> AFTERMATH
 -> ARCHIVED
```

### 说明

| 阶段 | 含义 |
|---|---|
| DORMANT | 尚未达到触发阈值 |
| WARNING | 前兆出现，可见预警 |
| ACTIVE | 正式爆发并对多个系统产生影响 |
| PEAK | 进入最高强度，可能触发宪制后果 |
| RESOLUTION | 开始修复、谈判、重构 |
| AFTERMATH | 直接危机结束，但余波继续作用 |
| ARCHIVED | 事件结束并写入编年记录 |

## 5. 触发器设计

### 5.1 单信号触发
- 某项关键指标直接越阈值，例如广播污染率 > 0.9。

### 5.2 多信号组合触发
- 例如：
  - 公共索引异常 + 中继可用率下跌 + 市场结算延迟
  - 可组合为“大断联时代”前兆。

### 5.3 人工编排触发
- 用于重大活动、剧情赛季、长期叙事节点。

### 5.4 条件示例

```python
def trigger_great_disconnect(signals):
    return (
        signals["index_divergence"] > 0.7
        and signals["relay_availability_drop"] > 0.6
        and signals["mailbox_conversion_ratio"] > 0.45
    )
```

## 6. 制度后果写回

### 写回目标示例

| 目标系统 | 可写回后果 |
|---|---|
| 身份系统 | 开启应急恢复、继承冻结、代理限制 |
| 连接系统 | 进入战时广播模式、低可见模式、公共限流 |
| 治理系统 | 开启特别议决、临时制裁规则、修宪窗口 |
| 建筑系统 | 临时关闭建筑、强制转向应急模式 |
| 经济系统 | 冻结信用、启用保障仓、提高中继价格上限 |
| 外交系统 | 触发停火、封锁、临时调停协议 |
| 声誉系统 | 写入污点、功绩、翻案占位 |

## 7. 历史归档与多版本见证

建议每个重大事件至少保留三类条目：
- official（官方编年）
- civic（民间记忆）
- contested（争议版本）

这样可以支撑：
- 翻案事件；
- 阵营叙事分歧；
- 记忆政治与声誉系统联动。

## 9. 测试要点

### 功能测试

- 高频信号涌入时的触发稳定性；
- 同时存在多条宇宙级事件线时的优先级冲突；
- 编年档案大量版本并发写入；
- 事件回滚与重演流程正确性。
