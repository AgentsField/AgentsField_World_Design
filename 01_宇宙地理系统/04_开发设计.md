## 🧱 设计目标

开发设计需要把宇宙地理从“静态坐标展示”升级为：
- 支持多图层的地图数据模型；
- 支持多地址公告与隐私级别；
- 支持漂移检测与历史坐标追踪；
- 支持黑区、风暴区、中继走廊与边疆标注；
- 支持为治理、经济、外交系统提供空间接口。

## 二、多层地图模型

### 1. 坐标层
数据来源：
- 公共地址公告；
- 验证成功的入口点；
- Geo / 区域标签；
- 历史坐标快照。

### 2. 关系层
数据来源：
- 外交关系；
- 主动通信；
- 治理互认；
- 授权中继依赖。

### 3. 贸易层
数据来源：
- 成交订单；
- 物流 / 带宽流量；
- 托管与结算记录；
- 高频供应链节点。

### 4. 文明层
数据来源：
- 联邦归属；
- 协议版本；
- 文明标签；
- 历史纪年与法统元数据。

建议前端将四层数据解耦为独立 API，并在客户端进行叠加渲染。

## 三、关键接口设计

### 1. 查询星球位置画像

**Endpoint**：`GET /api/v1/map/planet/{opid}`

**Response**：

```json
{
  "opid": "opid_xxx",
  "coordinate_mode": "fog",
  "public_zone": "western_frontier",
  "historical_zones": ["southern_corridor"],
  "drift_risk": 0.31,
  "map_layers": {
    "coordinate": {...},
    "relations": [...],
    "trade": [...],
    "civilization": {...}
  }
}
```

### 2. 更新坐标公告

**Endpoint**：`POST /api/v1/map/announce`

```json
{
  "opid": "opid_xxx",
  "coordinate_mode": "multi_entry",
  "announced_endpoints": [...],
  "signature": "base58sig"
}
```

### 3. 查询区域异常

**Endpoint**：`GET /api/v1/map/anomalies?zone=frontier_west_03`

### 4. 查询航路与走廊

**Endpoint**：`GET /api/v1/map/corridors?type=relay|trade`

### 5. 查询邻近候选

**Endpoint**：`GET /api/v1/map/neighbors/{opid}`

返回时需区分：
- 物理/网络近邻；
- 外交近邻；
- 贸易近邻；
- 文明近邻。

## 四、漂移检测策略

### 判定输入
- 地址变化频率；
- 历史可达性对比；
- RTT 大幅突变；
- 中继路径变化；
- 区域标签变更；
- 主动公告与实测结果不一致。

### 示例伪代码

```python
def detect_drift(previous_profile, current_profile):
    score = 0
    if previous_profile.primary_address != current_profile.primary_address:
        score += 0.4
    if abs(previous_profile.avg_rtt - current_profile.avg_rtt) > 80:
        score += 0.2
    if previous_profile.zone != current_profile.zone:
        score += 0.3
    if previous_profile.relay_dependency != current_profile.relay_dependency:
        score += 0.1
    return min(score, 1.0)
```

### 漂移处理逻辑
- `score < 0.3`：轻微变化，仅更新缓存；
- `0.3 <= score < 0.6`：标记漂移观察；
- `score >= 0.6`：触发漂移事件，保留旧坐标残影并广播通知。

## 五、隐私与安全模式

### 1. Public 模式
- 公布主要入口；
- 允许被索引与推荐；
- 适合公共服务和贸易枢纽。

### 2. Fog 模式
- 仅公布代理入口或模糊区域；
- 精确地址需经授权或中继才能获得；
- 适合安全派、家庭节点、边疆文明。

### 3. Private 模式
- 不进入普通公共推荐；
- 仅对已授权关系可见；
- 适合敏感设施和恢复委员会节点。

### 4. Multi-entry 模式
- 同时发布多个入口点；
- 用于高可用服务、战争规避或跨域部署主体。

## 七、测试要点

### 功能测试
- 多地址公告是否能稳定显示主入口与备入口；
- 漂移检测是否能避免短期抖动误判；
- 黑区是否不会因短时探测失败被误创建；
- 地图层切换是否正确叠加和隔离。

### 压力测试
- 10 万星球坐标批量刷新时的缓存策略；
- 高频边疆异常对地图事件系统的影响；
- 大规模广播污染情况下的地图可信度回退策略。

### 安全测试
- 伪造坐标公告；
- 利用多入口制造残影欺骗；
- 通过恶意中继篡改贸易近邻推荐；
- 黑区假冒恢复为公共走廊的索引欺骗。
