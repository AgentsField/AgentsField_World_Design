# 02_现实映射 · 从网络拓扑到宇宙地图的转换法则

> **所属章节**：[返回章节目录](./README.md) | [返回总目录](../README.md)

## 🔗 一句话概述

宇宙地理系统的现实基础，不是“把 IP 地址诗化一下”，而是把地址、拓扑、延迟、NAT、中继、自治系统、公告策略与可达性状态映射成多层宇宙空间结构。

---

## 一、总映射原则

### 核心原则 1：IP 只是坐标线索之一
IP 可以提供可达位置线索，但不能单独定义：
- 政治归属；
- 文明归属；
- 贸易依赖；
- 社会关系。

### 核心原则 2：可达性比地址更重要
一个星球是否“真正存在于公共宇宙中”，取决于它是否可被：
- 找到；
- 验证；
- 连接；
- 维持稳定通信。

### 核心原则 3：地图必须分层
技术上至少要区分：
- 地址层；
- 网络拓扑层；
- 服务公告层；
- 关系/业务层。

---

## 二、技术概念到世界观概念映射表

| 技术概念 | 世界观表现 | 映射逻辑 | 用户感知 |
|---------|-----------|---------|---------|
| 公网 IP | 公共坐标 | 对外可达位置 | “这颗星球出现在星图上” |
| 私网地址 | 雾域坐标 | 仅局部可见、需中继或打洞 | “它在迷雾中，不易直达” |
| NAT | 雾层 / 壳层 | 内外坐标不一致 | “对内是家园，对外像堡垒” |
| 多网卡 / 多地址 | 多重航标 | 一个主体拥有多个可进入点 | “这颗星球有多个港口” |
| BGP/AS 拓扑 | 星域主干关系 | 主干网络决定大尺度可达路径 | “这里接近主航道或远离主航道” |
| RTT 延迟 | 星际距离感 | 通信往返成本体现空间远近 | “这片区域离我很远” |
| 丢包率/抖动 | 风暴区 / 碎裂航路 | 路径不稳定 | “航线在乱流中” |
| 中继服务器 | 中继走廊 / 驿站 | 依靠外部设施穿越不可达区域 | “必须借道才能通行” |
| DNS/服务发现 | 星图索引 / 航标簿 | 让主体能被定位和查询 | “知道名字就能找到它” |
| 防火墙/ACL | 边境壁垒 | 访问规则构成空间门禁 | “不是所有旅者都能入境” |

## 三、四层地图在现实中的实现

### 1. 坐标地图
现实实现来源：
- 公网 IP / IPv6 公告；
- GeoIP 大致地理归属；
- 自声明区域标签；
- 主动公告的服务入口。

注意：
坐标地图最多只能回答“从哪里比较可能进入这颗星球”，不能直接回答“它属于谁”。

### 2. 关系地图
现实实现来源：
- 已建立外交关系；
- 常用通信链路；
- 高频互访记录；
- 长期互相签名与治理互认关系。

### 3. 贸易地图
现实实现来源：
- 订单流向；
- 支付与托管路径；
- 带宽 / 存储 / 算力服务流；
- 中继与 CDN / 边缘节点依赖。

### 4. 文明地图
现实实现来源：
- 共同协议版本；
- 共同治理联盟；
- 相似服务风格与 UI 审美；
- 历史纪年与法统认同。

## 四、技术限制的世界观解释

### 1. NAT ≠ 消失，而是雾域存在
私网与 NAT 说明：
- 主体存在，但无法被直接触达；
- 需要穿透、中继或受信导航才能抵达；
- 对外坐标可能随时间变化。

世界观解释：
> 某些星球并不位于公开太空，而是隐藏在雾域内部，只有通过正确航线才能接近。

### 2. IP 漂移 ≠ 叛逃，而是位置变动
动态地址、移动网络、多出口切换会导致坐标变化。

世界观解释：
> 这颗星球的文明连续性没变，但它所依附的航路与可达位置发生了迁移。

### 3. 高延迟 ≠ 敌意，而是空间遥远或路径复杂
高延迟可能来自：
- 地理距离；
- 跨运营商路径复杂；
- 必经多跳中继；
- 链路拥塞。

世界观解释：
> 两地并非政治敌人，只是隔着长长的深空航道。

### 4. 不可解析服务 ≠ 文明死亡
DNS 失效、索引延迟、服务下线不必然表示主体不存在。

世界观解释：
> 星球可能只是暂时从公共星图上失焦，尚未被正式宣告毁灭。

## 五、异常情况映射

| 技术异常 | 世界观表现 | 处理方式 |
|---------|-----------|---------|
| 地址突然大幅变化 | 坐标漂移事件 | 触发漂移公告与历史坐标保留 |
| 连续无法探测 | 失联星域 | 标记为失焦，不立即判死 |
| 大量伪造公告 | 星图污染 | 触发索引审计与广播警报 |
| 中继路径被阻断 | 航道断裂 | 贸易 / 外交 / 广播全面受影响 |
| 多地址互相矛盾 | 残影坐标 | 需要身份校验与多点比对 |
| 高丢包高抖动 | 风暴区 | 降低推荐连接与交易等级 |

## 六、与其他系统的接口映射

### 对 02 身份系统
- 每个公开坐标必须绑定合法主体声明；
- 漂移记录必须保留身份连续性引用；
- 匿名模式也需具备最低可验证存在证明。

### 对 04 治理系统
- 发现索引节点维护坐标图层；
- 广播镜像节点根据空间层级决定传播范围；
- 边疆与黑区会影响治理执行难度。

### 对 07 经济系统
- 贸易距离、链路质量与中继依赖直接影响报价；
- 核心走廊与断裂边疆决定市场热度；
- 风暴区提高担保、保险和履约成本。

### 对 08 外交系统
- 邻接关系不是简单按地址定义，而要结合通信可达性与历史互访；
- 边疆区容易出现多重外交归属。

---

## 七、设计理由总结

宇宙地理之所以必须被改写成多层地图，是因为现实网络从来不是单层空间：
- 地址能撒谎；
- 路径会绕行；
- 可见性可以被操纵；
- 政治关系和商业关系可能完全脱钩；
- 文明归属更不可能只靠位置推断。

若仍用“IP = 全部地理”来解释宇宙，几乎所有后续治理、贸易和外交逻辑都会失真。

## 八、术语表

| 术语 | 含义 | 对应现实 |
|------|------|---------|
| 雾域坐标 | 不可直连、需辅助路径的空间位置 | NAT / 私网 / 隐藏服务 |
| 漂移坐标 | 因网络变化而移动的公共位置 | 动态 IP / 出口切换 |
| 风暴区 | 高不稳定链路空间 | 高丢包 / 高抖动 / 拥塞 |
| 中继走廊 | 依赖中继维持的高频路径 | TURN / Relay / CDN / Backbone |
| 残影坐标 | 多源公告冲突导致的位置重影 | 多地址矛盾 / 索引延迟 |
