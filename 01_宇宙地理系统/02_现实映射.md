### 映射设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **直观性** | 用户一眼就能理解 | IP段对应星域，一眼看出所属大区 |
| **一致性** | 所有地方规则相同 | 所有IPv4星球都按相同规则划分 |
| **可预测** | 用户能预判结果 | 看到IP就能说出大概星域位置 |
| **有意义** | 映射有实际意义 | 同星域星球延迟更低 |
| **可扩展** | 支持新技术 | IPv6映射兼容IPv4逻辑 |

## 🌐 网络状态 → 空间层次映射

### 三态空间转换表

| 网络技术状态 | 世界观空间 | 视觉表现 | 连接能力 | 发现难度 |
|-------------|-----------|---------|---------|---------|
| 公网IPv4 | 现实空间 | ☀️明亮光芒 | 直接连接 | 容易发现 |
| 公网IPv6 | 现实空间 | ☀️明亮光芒（带蓝色光晕） | 直接连接 | 容易发现 |
| 双栈公网 | 双栖空间 | 🌟双重光环 | 双路径连接 | 非常容易发现 |
| NAT内网 | 迷雾空间 | 🌫️朦胧光晕 | 需穿透/中继 | 中等难度 |
| CGNAT | 浓雾空间 | 🌫️🌫️浓密迷雾 | 必须中继 | 较难发现 |
| 完全离线 | 虚空空间 | ⚫完全黑暗 | 无法连接 | 无法发现 |
| 仅局域网 | 内域空间 | 🏠家庭图标 | 仅限本地 | 不可发现 |

### 状态转换触发条件

```
空间层次转换流程：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

现实空间 → 迷雾空间
  触发：进入NAT网络、IP变成私有地址
  表现：光芒变暗，出现光晕
  过渡时间：即时

迷雾空间 → 现实空间  
  触发：获得公网IP、配置端口映射成功
  表现：光芒变亮，光晕消失
  过渡时间：数秒到数分钟

现实/迷雾空间 → 虚空空间
  触发：断网、关机、主动离线
  表现：光芒消失，变为黑点或隐藏
  过渡时间：数秒（心跳超时）

虚空空间 → 现实/迷雾空间
  触发：重新上线、恢复网络
  表现：重新发光，位置可能改变（漂移）
  过渡时间：数秒到数分钟

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 连接失败映射

| 技术错误 | 世界观表现 | 用户提示 | 解决建议 |

| 拒绝连接 | 虫洞入口关闭 | "对方星球拒绝访问" | 检查防火墙或外交关系 |

| 证书错误 | 主权印章验证失败 | "身份验证失败" | 检查密钥或重新建交 |

### 双栈的进化隐喻

IPv4→IPv6的迁移被视为"**星球进化**"：

```
进化阶段：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第一阶段：单栈星球
  IPv4-only或IPv6-only
  → 只能在一个宇宙中活动
  → 存在连接限制

第二阶段：双栖星球
  同时支持IPv4和IPv6
  → 可以在两个宇宙中自由穿梭
  → 获得双重机会和资源
  → 被视为"进化完成"的标志

第三阶段：原生星球
  IPv6-only（当IPv4退役后）
  → 完全生活在新宇宙中
  → 享受无限地址空间
  → 摆脱旧宇宙的束缚

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
