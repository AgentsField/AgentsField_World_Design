# 06 · 建筑与功能系统

> **核心映射**：软件服务、节点模块、能力编排与公共设施 → 星球上的建筑群、职能区与文明器官  
> **章节定位**：定义“文明如何把抽象能力建造成可见空间，以及一个星球究竟靠哪些建筑活着、治理、生产、防御与记忆”。

## 📖 章节概述

建筑与功能系统在修订后，不再把服务模块简单写成“建筑皮肤”，而是重写为**文明能力的空间化表达**。建筑不只是 UI 隐喻，而是把身份、治理、通信、资源、记忆、声誉与防御变成可观察、可部署、可升级、可争夺的功能器官。

本章的关键重写方向包括：
- 把建筑区分为主权建筑、公共建筑、生产建筑、记忆建筑、防御建筑与灰色建筑；
- 强调建筑不是装饰，而是承载真实服务能力、权限边界与维护成本；
- 让建筑之间形成依赖链：没有能源与存储，档案院无法工作；没有身份与治理支持，交易所只是危险空壳；
- 引入“功能过载、服务断层、建筑被劫持、文明废墟重启”等压力场景；
- 使 06 章成为 07 章资源经济与 09 章声誉成就的真实落点。

在 AgentsField 中，一颗星球之所以能被称为“文明”，并不是因为它拥有宏大叙事，而是因为它能把抽象协议建造成一组真正能运行、能失灵、能维修、能扩张的建筑器官。

## 🏗 核心概念速查

| 概念 | 简述 | 关键边界 |
|------|------|---------|
| 主权建筑 | 与身份、授权、根控制相关的核心设施 | 不能轻易外包 |
| 公共建筑 | 对外提供治理、广播、接待与结算服务的设施 | 受公共规则约束 |
| 生产建筑 | 算力、存储、带宽与加工能力的来源 | 直接影响经济产出 |
| 记忆建筑 | 档案、日志、历史与备份的承载体 | 影响连续性与恢复能力 |
| 防御建筑 | 防火墙、蜜罐、观察塔、隔离区等 | 不只是战斗，也包含安全与韧性 |
| 灰色建筑 | 黑市驿站、影子中继、非法铸造所等边缘设施 | 提供机会也带来风险 |
| 建筑链 | 建筑之间的依赖、授权与供给关系 | 决定文明是否真正可持续 |

## 🎯 本章回答的关键问题

1. **为什么服务要被写成建筑？**  
   因为建筑化能让抽象能力获得空间、成本、维护与损毁逻辑，文明才有真实质感。

2. **建筑是否只是隐喻？**  
   不是。每一栋建筑都应对应一类真实服务能力、权限要求与资源消耗。

3. **为什么一颗星球可能“有市场没文明”？**  
   因为若缺少主权建筑、记忆建筑与治理建筑，它只能短期繁荣，无法长期延续。

4. **为什么废墟重要？**  
   因为服务停摆后留下的并非空白，而是可被重启、掠夺、继承或考古的文明残骸。

5. **为什么本章是全局中枢章节之一？**  
   因为身份要落入主权殿堂，治理要依附公共大厅，经济要建立交易建筑，声誉和成就也必须附着于具体设施与历史遗存。

## 🔁 核心建筑闭环

### 1. 建造闭环
资源准备 → 权限校验 → 施工部署 → 对外公告 → 投入运行

### 2. 运维闭环
监控 → 负载变化 → 扩容 / 降级 → 故障修复 → 状态归档

### 3. 灾损闭环
攻击 / 过载 / 断供 → 局部停摆 → 紧急接管 → 修复或废弃 → 废墟再利用 / 重建

### 4. 文明升级闭环
建筑谱系完善 → 功能耦合增强 → 公共能力提升 → 声誉增长 → 吸引更多关系与资源 → 新一轮扩建

## ⚠ 结构难点摘要

| 难点 | 风险 | 本章处理方式 |
|------|------|-------------|
| 建筑只是装饰 | 与系统机制脱节 | 强化每类建筑的真实功能与成本 |
| 服务关系太平面 | 无法体现文明复杂度 | 用建筑链表达依赖与脆弱性 |
| 没有废墟与损毁逻辑 | 世界过于理想化 | 引入损毁、劫持、重启与废墟叙事 |
| 没有灰色建筑 | 缺少边缘冲突与黑暗面 | 补入黑市、中继黑塔、影子工坊 |
| 建筑与其他章节脱节 | 成为空洞美术层 | 让 02/04/07/09 均需通过建筑落地 |
