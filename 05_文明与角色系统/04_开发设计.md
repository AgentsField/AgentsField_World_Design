## 4. 角色授权与撤销流程

### 4.1 任命流程
1. 身份系统验证授权人合法性；
2. 检查是否具备可委派的制度权限；
3. 创建 InstitutionalRoleAssignment；
4. 向治理日志与声誉系统广播角色变更；
5. 视角色类型开放 API scope 与 UI 入口。

### 4.2 撤销流程
1. 验证撤销权来源；
2. 检查是否触发继承、交接或争议冻结；
3. 关闭 assignment；
4. 记录撤职原因；
5. 若涉及公共职责，通知治理、外交与资源系统更新可执行主体。

### 4.3 继承流程
- 领主缺席 / 永久失效；
- 存在预设继承链；
- 若存在争议则转入治理与见证程序；
- 文明档案中的“合法代表主体”字段更新。

## 5. 文明原型判定与混型逻辑

### 5.1 原型评分输入
- 资源结构
- 建筑使用比例
- 外交姿态
- 治理形式
- 声誉维度
- 关键创伤

### 5.2 判定伪代码

```python
def classify_civilization(metrics):
    scores = {
        "cartographer": metrics.archive_weight + metrics.audit_weight,
        "fortress": metrics.security_weight + metrics.isolation_weight,
        "bridge": metrics.relay_weight + metrics.connectivity_weight,
        "market_federation": metrics.trade_weight + metrics.credit_weight,
        "agent_civic_league": metrics.agent_rights_weight + metrics.co_governance_weight,
    }
    ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return ranked[0][0], ranked[1][0]
```

## 6. 阶层与派系图谱

建议使用图结构存储：

- 节点：文明、角色、派系、关键主体、关键资源；
- 边：授权、依赖、对立、担保、从属、竞争、继承；
- 权重：影响力、资源量、持续时长、敌对强度。

### 查询示例
- 查某文明当前最有影响力的派系；
- 查某治理代理同时控制哪些资源入口；
- 查某文明中“高资源控制但低公共声誉”的隐性权力节点；
- 查某 Agent 是否拥有可见公共人格记录与申诉资格。

## 8. 与其他系统的对接

| 系统 | 对接内容 |
|---|---|
| 身份系统 | 验证主体、代理资格、继承合法性 |
| 治理系统 | 角色变更日志、议席资格、见证程序 |
| 建筑系统 | 不同角色与派系对建筑控制权 |
| 经济系统 | 资源控制与阶层形成关系 |
| 外交系统 | 文明原型、对外姿态、联盟成员身份 |
| 声誉系统 | 角色行为、治理功绩、污点追踪 |
| 事件系统 | 内战、继承战、平权运动与派系分裂事件 |

## 9. 测试要点

### 功能测试
- 任命 / 撤销 / 继承流程正确性
- 文明原型自动判定与混型切换
- 成员类型转换（客民→居民→治理代理）
- 派系图谱查询性能

### 风险测试
- 非法代理滥用任命权限
- 领主失联下的多继承链冲突
- 派系伪装为官方治理角色
- Agent 公共人格档案被恶意抹除或伪造
