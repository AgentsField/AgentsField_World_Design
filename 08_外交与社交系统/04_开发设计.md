## 3. 外交状态机

```text
DISCOVERED
  -> CONTACTED
  -> RECOGNIZED
  -> MUTUAL_ACCESS
  -> TREATY_BOUND
  -> ALLIED
  -> SANCTIONED
  -> SEVERED
  -> BLOCKADED
  -> PROXY_CONFLICT
```

### 状态规则

| 状态 | 含义 | 典型触发 |
|---|---|---|
| DISCOVERED | 知晓存在 | 公共索引或引荐出现 |
| CONTACTED | 已发起接触 | 首次可识别信号或函件 |
| RECOGNIZED | 正式建交 | 双方承认合法主体地位 |
| MUTUAL_ACCESS | 开放部分接口 | 通讯、市场或目录开放 |
| TREATY_BOUND | 已签条约 | 至少一项正式协定生效 |
| ALLIED | 形成稳定共同体协作 | 防御、贸易、治理或中继同盟 |
| SANCTIONED | 公共权利受限 | 违约、污染、封锁前置 |
| SEVERED | 断交 | 正式终止互认 |
| BLOCKADED | 系统性隔离 | 多层限制并行 |
| PROXY_CONFLICT | 进入灰区对抗 | 不公开开战但持续对抗 |

## 6. 执行动作与联动接口

### 6.1 与连接系统联动
- RECOGNIZED 才允许某些会话请求升级；
- SANCTIONED 会降低或关闭公共中继优先级；
- BLOCKADED 会直接写入连接策略黑名单。

### 6.2 与经济系统联动
- 条约决定关税、准入、担保规则；
- 制裁冻结市场权限；
- 联盟可共享部分带宽或中继补贴。

### 6.3 与治理系统联动
- 争端进入见证与仲裁流程；
- 多方制裁需满足议决门槛；
- 联盟裂解可触发治理重构。

### 6.4 与声誉系统联动
- 履约提高外交信誉；
- 违约生成污点；
- 长期中立调停可生成外交成就。

## 8. 测试要点

### 功能测试
- 建交、互认、签约、联盟成立的状态流正确性；
- 条约条款对连接与经济权限的联动生效；
- 制裁与封锁的局部 / 全局作用域正确；
- 代理冲突记录可被隐式追踪但不误判为全面战争。

### 风险测试
- 伪造条约签名；
- 多边联盟中制裁执行不一致；
- 长期冷暴力状态对用户体验与系统逻辑的影响；
- 高中心性节点断交对图谱稳定性的冲击。
