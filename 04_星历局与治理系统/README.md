# 04 · 星历局与治理系统

> **核心映射**：公共索引、广播、仲裁记录与协议执行网络 → 宇宙共同体的有限权力治理结构  
> **章节定位**：定义“谁来见证公共秩序、谁来执行公共流程、公共权力的边界在哪里”。

## 📖 章节概述

星历局在修订后不再被写作一个温和而模糊的“中立档案馆”，而是一个**协议化、可审计、多中心、有限权力的公共秩序网络**。它不是宇宙君主，不拥有主权源头，但它确实承担公共索引、广播见证、争议记录、治理执行与联邦修复等关键职责。

本章的核心改写包括：
- 将星历局拆解为五类节点：注册见证、发现索引、广播镜像、仲裁记录、治理执行见证；
- 明确其权力边界：不能剥夺根印、不能解密私密内容、不能单方删除身份存在；
- 建立双院制治理：星球院维护主权平等，贡献院维护公共建设现实；
- 补齐提案、审议、投票、执行、上诉、分裂与重构的完整治理闭环；
- 让危机状态具备制度出口，例如广播污染、索引争夺、联邦裂变、执行拒绝与节点失信。

简言之：星历局不再只是“看见世界”，而是负责让共同体在不摧毁主权的前提下继续运转。

## 🏛 核心概念速查

| 概念 | 简述 | 边界说明 |
|------|------|---------|
| 注册见证节点 | 负责记录新主体进入公共宇宙 | 只见证，不授予根本主权 |
| 发现索引节点 | 维护可查询、可达的公共目录 | 能隐藏索引，不等于抹除实体 |
| 广播镜像节点 | 中继与见证公共信息传播 | 不应解读私密正文 |
| 仲裁记录节点 | 保存争议证据、裁决与上诉链 | 记录不等于永恒正确 |
| 治理执行见证节点 | 对集体决议的执行状态进行公示 | 不能代替全体主权主体直接统治 |
| 星球院 | 按主体平等参与的治理机构 | 防止大文明以体量碾压小星球 |
| 贡献院 | 按公共建设贡献形成的治理机构 | 防止“只享权利不建共同体” |
| 有限公共权力 | 可限制接入、索引、结算、表决资格 | 不可抹除本地主权存在 |

## 🎯 本章回答的关键问题

1. **为什么需要星历局？**  
   因为没有公共索引、协议见证与争议记录，宇宙将重新坠回发现失序、广播污染与治理失效的黑暗期。

2. **星历局是否会成为新中心化霸权？**  
   本章的答案是：它有这种风险，因此必须被设计为多中心、开源、可审计、可分裂、可替换的有限权力网络。

3. **谁来代表公共利益？**  
   不是单一机构，而是双院治理、节点公开日志、上诉程序与联邦竞争共同构成的动态平衡。

4. **治理如何不压碎主权？**  
   通过区分“存在权”与“公共服务权”：星历局可限制参与资格、广播优先级、索引露出、结算接入，但不能抹去主体的根本存在。

5. **当星历局自身出错时怎么办？**  
   文档已补齐裂局、重编年、索引分叉、见证失败与联邦恢复机制。

## 🔁 核心治理闭环

### 1. 公共接入闭环
注册请求 → 见证校验 → 索引入册 → 广播告知 → 状态维护

### 2. 提案治理闭环
提案提交 → 双院审议 → 投票通过 → 执行见证 → 结果公示 → 上诉复核

### 3. 争议处理闭环
争议报案 → 证据冻结 → 仲裁记录 → 临时措施 → 正式裁决 → 申诉或执行完结

### 4. 分裂恢复闭环
节点失信 / 联邦裂变 → 镜像接管 → 编年分叉 → 最小秩序恢复 → 重建共识 → 新秩序合流或并存

## ⚠ 技术与叙事难点摘要

| 难点 | 风险 | 本章处理方式 |
|------|------|-------------|
| 星历局权力过于神化 | 变成事实上的皇权中心 | 明确五类节点与禁止权力清单 |
| 治理只写理念不写执行 | 无法支撑冲突剧情与产品规则 | 补齐提案-审议-执行-上诉状态流 |
| 联邦与主权冲突模糊 | 无法解释制裁边界 | 引入存在权 / 公共权利二分 |
| 节点失信无代价 | 公共秩序不可验证 | 建立日志审计、轮值、削权与替换机制 |
| 广播与索引遭污染 | 世界缺乏敌意环境 | 引入污染事件、分叉事件与恢复协议 |
