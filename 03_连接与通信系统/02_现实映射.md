🌉 中继虫洞（Relay Wormhole）
─────────────────────────────────────────
技术基础：TURN协议 + WebSocket中继
实现条件：
  ⚠️ 双方都无法直接连接
  ✓ 存在可用的TURN服务器（中继星球）
  ✓ 中继星球有公网IP和充足带宽

中继流程：
  1. A → TURN服务器：分配中继地址
  2. TURN服务器 → A：返回 relay.af:30001
  3. B → TURN服务器：分配中继地址
  4. TURN服务器 → B：返回 relay.af:30002
  5. A ↔ TURN服务器 ↔ B：数据中转

数据流向：
  A发送 → TURN:30001 → TURN内部转发 → TURN:30002 → B接收

隐私保护：
  ✓ TURN服务器只能看到加密后的数据包
  ✓ 无法解密内容（无会话密钥）
  ✓ 无法确定通信双方身份（仅知道IP）

性能影响：
  - 延迟增加：约+20-100ms（取决于中继位置）
  - 带宽占用：双倍流量（上传+下载）
  - 依赖可用性：中继故障则通信中断

世界观表现：
  → 两段光柱，中间有中转节点
  → 中转节点显示"中继星球"图标
  → 整体亮度较低，表示间接连接
  → 需要支付"中继服务费"

    // 验证MAC
    expected_mac = hmac_sha256(authentication_key, encrypted_msg.payload)

    // 解密
    nonce = base64_decode(encrypted_msg.nonce)
    plaintext = secretbox_decrypt(
      key=encryption_key,
      ciphertext=base64_decode(encrypted_msg.payload),

## 🌫️ NAT穿透技术完整映射

### NAT类型世界观映射

| NAT类型 | 技术特征 | 世界观表现 | 穿透难度 | 推荐策略 |
|---------|---------|-----------|---------|---------|
| **公网IP** | 无NAT | ☀️现实空间 | 无需穿透 | 直接连接 |
| **全锥型(Full Cone)** | 任意外部可向内发送 | 🌫️薄雾空间 | ⭐ 极易 | STUN后直接连接 |
| **地址限制型** | 仅已向之外发过的地址可进入 | 🌫️🌫️中等迷雾 | ⭐⭐ 较易 | STUN + 预连接 |
| **端口限制型** | 仅特定端口可进入 | 🌫️🌫️🌫️浓密迷雾 | ⭐⭐⭐ 中等 | STUN + 端口预测 |
| **对称型(Symmetric)** | 每目标分配不同映射 | 🌫️🌫️🌫️🌫️极浓迷雾 | ⭐⭐⭐⭐⭐ 极难 | 必须使用TURN中继 |
| **双重NAT** | 多层NAT嵌套 | 🌀迷雾迷宫 | ⭐⭐⭐⭐ 很难 | 双重穿透或中继 |

### 虫洞通信的哲学意义

```
虫洞即自由：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

在传统互联网中：
  消息路径：A → 服务器 → B
  问题：服务器可以看到所有内容
  问题：服务器可以审查、延迟、修改
  问题：服务器是单点故障

在AgentsField虫洞中：
  消息路径：A ⟷ B（直接）
  优势：只有双方能看到内容
  优势：无法被审查（数学保护）
  优势：去中心化，无单点控制

虫洞三定律的深层含义：
  直连定律 = 消除中间人控制
  加密定律 = 保护通信隐私权
  主权定律 = 自主决定与谁通信

这就是数字主权的通信实现。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
