### 资源计量系统

```
资源计量架构：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

计量器设计：

算力计量：
  - 每秒采样CPU使用率
  - 累加计算总消耗
  - 公式：算力消耗 = Σ(CPU% × 核心数 × 时间)
  
  示例：
    2核CPU，50%使用率运行1小时
    算力消耗 = 0.5 × 2 × 1 = 1算力小时

存储计量：
  - 每小时采样存储占用
  - 计算存储时间积分
  - 公式：存储消耗 = 平均占用 × 时间
  
  示例：
    500GB存储使用24小时
    存储消耗 = 500 × 24 = 12000 GB小时

带宽计量：
  - 实时统计流量
  - 区分入站/出站
  - 公式：带宽消耗 = 传输字节数
  
  示例：
    传输10GB数据
    带宽消耗 = 10GB（出站）

数据模型：
  ResourceUsage: {
    planet_opid: OPID,
    resource_type: "compute|storage|bandwidth",
    amount: Decimal,
    unit: "unit|GB|GB_hours|GB_transfer",
    period_start: Timestamp,
    period_end: Timestamp,
    metadata: JSON
  }

## 🔄 交易系统技术实现

### 交易撮合引擎

```
交易系统架构：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

撮合引擎设计：

订单簿结构：
  OrderBook: {
    service_type: "storage|compute|bandwidth",
    bids: [  // 买单（购买服务）
      {price: 10, amount: 100, planet_opid: "opid_A"},
      {price: 9, amount: 200, planet_opid: "opid_B"}
    ],
    asks: [  // 卖单（提供服务）
      {price: 11, amount: 150, planet_opid: "opid_C"},
      {price: 12, amount: 300, planet_opid: "opid_D"}
    ]
  }

撮合算法：
  function match_orders():
    while bids not empty and asks not empty:
      best_bid = highest_price(bids)
      best_ask = lowest_price(asks)
      
      if best_bid.price >= best_ask.price:
        // 成交
        trade_amount = min(best_bid.amount, best_ask.amount)
        trade_price = (best_bid.price + best_ask.price) / 2
        
        execute_trade(best_bid, best_ask, trade_amount, trade_price)
        
        // 更新订单簿
        update_order_book(best_bid, best_ask, trade_amount)
      else:
        break  // 无法继续撮合

交易流程：
  1. 卖方发布服务供应订单（Ask）
  2. 买方发布服务需求订单（Bid）
  3. 系统撮合匹配订单
  4. 双方确认交易
  5. 服务交付
  6. 资金结算
  7. 评价反馈
