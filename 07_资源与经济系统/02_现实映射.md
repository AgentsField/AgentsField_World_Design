# 02_现实映射 · 从硬件能力到服务承诺经济

> **所属章节**：[返回章节目录](./README.md) | [返回总目录](../README.md)

---

## 1. 文档目标

本文件不再只做“CPU 对应算力、磁盘对应存储”的静态换算，而是解释：

> 世界观中的资源、市场、履约、托管、担保、StarCoin，在现实系统里分别对应什么。

修订后的现实映射必须覆盖五层：

1. 物理能力层；
2. 可调度资源层；
3. 服务承诺层；
4. 结算与担保层；
5. 争议与仲裁层。

---

## 2. 总映射框架

```text
世界观：算力 / 存储 / 带宽 / 信任
→ 现实：硬件能力 / 网络能力 / 履约能力 / 信誉与担保能力

世界观：交易
→ 现实：服务合约 + 托管支付 + 履约证明 + 验收 + 仲裁

世界观：StarCoin
→ 现实：结算资产 + 治理押金 + 风险担保工具
```

---

## 3. 基础资源映射

| 世界观资源 | 现实对应 | 关键指标 |
|-----------|---------|---------|
| 算力 | CPU/GPU/NPU/集群可调度能力 | 峰值、占用率、任务时长、完成量 |
| 存储 | SSD/HDD/对象存储/冗余层 | 容量、持续时间、可用性、完整性 |
| 带宽 | 上下行能力、转发能力、中继能力 | 速率、延迟、丢包、持续性 |
| 信任 | 履约记录、争议率、担保能力、治理信誉 | 违约率、历史时长、公共评分、保证金水平 |

关键修正是：

现实系统交易的不是“硬件本身”，而是**硬件所能在一定时间窗口内稳定交付的服务能力**。

---

## 4. 资源不等于服务

### 4.1 旧映射问题
旧版常把：
- 1 台高端 GPU；
- 1TB 存储；
- 100Mbps 带宽；

直接当成可售资源。

### 4.2 新映射原则
现实上必须区分：

#### 物理峰值能力
设备在理想条件下的最大能力。

#### 可调度能力
当前真实可分配给外部订单的能力。

#### 可承诺能力
在指定时间段内，愿意并且能够签约承担的能力。

#### 可验证交付能力
不仅能做，还能拿出证据证明你真的做了。

---

## 5. 服务承诺的现实结构

世界观中的“服务承诺”，现实上对应一份结构化订单对象。

```json
{
  "order_type": "compute|storage|bandwidth|relay|archive",
  "promised_capacity": {
    "value": 8,
    "unit": "gpu_hours"
  },
  "delivery_window": {
    "start": "2026-03-07T00:00:00Z",
    "end": "2026-03-08T00:00:00Z"
  },
  "quality_requirements": {
    "availability": 0.99,
    "max_latency_ms": 80,
    "proof_required": true
  },
  "price": {
    "amount": 240,
    "currency": "STAR"
  }
}
```

这意味着世界观里的交易对象，现实上更接近：
- SLA 化服务；
- 可证明任务执行；
- 时间约束下的能力租赁；
- 风险可担保的订单。

## 6. 履约证明映射

这是旧版缺失最严重的一层。

### 6.1 计算服务证明
现实可对应：
- 作业日志；
- 任务完成摘要；
- 可验证任务哈希；
- 运行时间与资源使用记录。

### 6.2 存储服务证明
现实可对应：
- PoR（Proof of Retrievability）；
- PoS（Proof of Storage）；
- 分块哈希校验；
- 随机挑战响应。

### 6.3 带宽 / 中继证明
现实可对应：
- 流量计量；
- 会话记录摘要；
- 延迟与丢包日志；
- 双边确认的转发计费记录。

### 6.4 为什么这层关键
没有履约证明，就没有：
- 托管自动释放；
- 仲裁基础；
- 大规模市场；
- 高信任文明经济。

---

## 7. 托管与结算映射

### 世界观表达
托管支付、分期释放、冻结、赔偿。

### 现实对应
- escrow account；
- smart contract escrow；
- staged settlement；
- dispute freeze。

### 典型流程
```text
买方下单
→ 资金进入托管
→ 卖方交付服务
→ 上传履约证明
→ 买方验收 / 自动验收
→ 资金释放
→ 如争议则冻结并进入仲裁
```

因此世界观中的“市场秩序”，现实上对应一套：
- 支付前锁定；
- 履约后释放；
- 争议时冻结；
- 裁决后再分配。

---

## 8. StarCoin 映射

## 8.1 作为结算资产
现实上类似：
- 原生平台清算代币；
- 跨主体服务交易媒介；
- 可编程支付资产。

## 8.2 作为治理押金
现实上类似：
- proposal bond；
- validator stake；
- misconduct slashable stake。

## 8.3 作为信用担保工具
现实上类似：
- 保证金；
- 风险抵押品；
- 赔付准备金；
- 保险池注资资产。

### 关键边界
如果 StarCoin 只有转账功能，它就只是装饰币；
只有同时承担**结算 + 押金 + 担保**三重作用，它才是制度性货币。

---

## 9. 信任资源映射

世界观里新增的“信任”资源，现实上不能只理解为口碑。

它更接近：
- 信用评分；
- 保证金折扣资格；
- 高风险订单准入资格；
- 联邦公共岗位可信度；
- 保险池接纳程度。

### 映射逻辑
高信任主体现实上拥有：
- 更低的交易摩擦；
- 更低的保证金要求；
- 更高的报价权；
- 更大的公共角色空间。

所以信任不仅是结果，也是可被资本化的生产要素。

---

## 10. 黑市与灰市映射

### 世界观表达
影子结算层、地下市场、规则外交易。

### 现实对应
- 场外结算；
- 私下担保；
- 非公开合同；
- 未登记资源租用；
- 非公共仲裁。

### 为什么会出现
现实里只要：
- 公共手续费过高；
- 准入要求过重；
- 仲裁太慢；
- 制裁太严；

就会自然催生更快、更隐蔽、但更高风险的影子市场。

---

## 11. 寡头与反垄断映射

### 世界观表达
算力寡头、带宽霸权、桥梁文明控制通路。

### 现实对应
- 大型集群运营商；
- 网络骨干控制者；
- 市场流量入口平台；
- 高信誉垄断服务商。

### 风险机制
当规模收益、信誉收益、基础设施依赖三者叠加时，市场会自然走向集中。

### 修订后的现实边界
因此系统设计不能只做自由市场映射，还必须加入：
- 集中度监测；
- 大节点披露义务；
- 高依赖风险标记；
- 反操纵规则；
- 对新节点的冷启动扶持。

## 12. 与其他系统接口

## 12.1 与身份系统
经济系统需要身份系统提供：
- 授权链；
- 委托范围；
- 被盗或争议状态。

## 12.2 与治理系统
经济系统需要治理层提供：
- 市场准入状态；
- 制裁状态；
- 节点资格变化。

## 12.3 与声誉系统
经济系统需要信誉层提供：
- 履约评分；
- 争议历史；
- 风险等级；
- 赔偿记录。

## 12.4 与连接系统
带宽、中继、延迟、链路质量会直接成为可出售服务与履约证明的一部分。

## 13. 一张总图：经济现实分层

```text
第 1 层：物理能力层
- CPU / GPU / Disk / Network

第 2 层：可调度资源层
- available capacity / reserved capacity / measured capacity

第 3 层：服务承诺层
- order / SLA / commitment / delivery window

第 4 层：结算与担保层
- escrow / settlement / collateral / insurance pool

第 5 层：争议与制度层
- proof / acceptance / arbitration / penalty
```
