## 1. 文档目标

本文件用于回答一个关键问题：

> AgentsField 世界观中的“主权印章、领主、星球身份、代理人、继承、恢复”，在现实技术与制度层分别对应什么？

修订后的重点不再是把密码学术语简单诗化，而是建立四层映射：

1. **密码学身份层**：证明你是谁；
2. **授权控制层**：规定你能做什么；
3. **公共记录层**：记录你曾做过什么；
4. **制度合法性层**：判断你的行为是否被共同体承认。

---

## 2. 总映射框架

```text
世界观层
→ Root Seal / Planet ID / Runtime Key / Agent Persona / Succession

现实技术层
→ Root Key / DID-like Identifier / Session Key / Capability Token / Recovery Policy

制度层
→ 授权模型 / 恢复规则 / 继承程序 / 仲裁记录 / 公共状态标记

公共叙事层
→ 领主 / 星球 / 代理人 / 失落星球 / 争议继承体
```

## 3. 关键术语映射表

| 世界观术语 | 现实映射 | 说明 |
|-----------|---------|------|
| 主权印章 Root Seal | 根密钥 / 多签根身份 | 最高主权锚点，低频使用 |
| OpenPlanetID / Planet ID | 长期公共标识（DID-like ID / 哈希标识） | 跨时间持续识别星球 |
| 运行身份 Runtime Key | 会话签名密钥 / 服务密钥 | 用于日常签名、服务调用、握手 |
| Agent Persona | 受限子身份 / capability-bound principal | 带边界的执行人格 |
| 公共信誉身份 | 公共信誉档案 / reputation record | 不是认证，而是社会评价 |
| 领主委托 | Delegation / capability grant | 有边界、有期限、可撤销的授权 |
| 主权继承 | Succession policy + dispute resolution | 不是密码学自动完成，而是制度程序 |
| 身份恢复 | Recovery policy + witness confirmation | 技术恢复与共同体见证的结合 |
| 争议星球 | Contested identity state | 公共标记状态，不等于删除主体 |
| 封存星球 | Archived / sealed identity state | 失联或未决主体的保留状态 |

## 4. 五层身份在现实中的对应

## 4.1 Root Seal

### 世界观表达
领主持有的最终主权印章，用于恢复、继承、最终授权与紧急撤销。

### 现实对应
- Ed25519 根密钥；
- 多签阈值密钥；
- 硬件安全模块中的主控制密钥；
- 或多人共同掌控的 root authority set。

### 现实边界
Root Seal 在技术上只能证明：
- 某个主权根能够签署声明；
- 某项授权来自主权根；
- 某次恢复/继承声明有根层支持。

但它不能独自证明：
- 该声明一定被共同体承认；
- 没有发生盗印；
- 没有更高优先级的争议程序正在进行。

## 4.2 Planet ID

### 世界观表达
星球跨时间延续的公共身份。

### 现实对应
- 由公钥、声明材料或根身份派生出的长期标识；
- 类似 DID、主体哈希、不可变公开身份索引。

### 现实边界
Planet ID 的作用是唯一识别，不是自动赋权。它回答“哪个主体”，不回答“该主体当前具备哪些公共权利”。

## 4.3 Runtime Key

### 世界观表达
星球日常运行层的签名能力。

### 现实对应
- Session signing key；
- TLS/client auth key；
- service key pair；
- transport encryption key。

### 现实边界
Runtime Key 应当被视为可替换运行部件，而不是主权神体。它适合高频操作，不适合承担终局合法性。

## 4.4 Agent Persona

### 世界观表达
受限执行人格。

### 现实对应
- service account；
- delegated principal；
- scoped API credential；
- capability token subject。

### 现实边界
Agent Persona 的核心不是“像人”，而是“权限有边界”。它必须可撤销、可审计、可追责。

## 4.5 Reputation Profile

### 世界观表达
公共社会人格。

### 现实对应
- reputation ledger；
- public service score；
- dispute history；
- contract fulfillment history。

### 现实边界
信誉不能替代认证，更不能替代主权。它只能影响：
- 推荐排序；
- 市场溢价；
- 治理门槛；
- 公共信任阈值。

## 5. 认证 / 授权 / 信誉 / 合法性的现实分层

## 5.1 Authentication

### 现实对应
- 公私钥签名验证；
- challenge-response；
- OPID 与公钥匹配校验。

### 回答
“当前发言者是不是声明中的身份主体。”

## 5.2 Authorization

### 现实对应
- ACL；
- RBAC；
- ABAC；
- capability token；
- scoped delegation document。

### 回答
“该主体当前被允许做什么。”

## 5.3 Reputation

### 现实对应
- 历史履约评分；
- 争议记录；
- 服务稳定性指标；
- 公共贡献记录。

### 回答
“该主体是否值得信任。”

## 5.4 Legitimacy

### 现实对应
- 继承程序是否合法；
- 恢复是否满足 witness threshold；
- 当前是否处于争议期；
- 仲裁记录是否认定该主张成立。

### 回答
“该主体此刻的行动，是否被制度承认。”

## 6. 从单纯密码学到制度化身份

早期设计最大的误区，是把密码学与制度混为一谈。

### 旧逻辑
```text
能签名 = 是身份持有者 = 有全部权力 = 有全部正当性
```

### 新逻辑
```text
能签名
= 通过认证
≠ 自动获得授权
≠ 自动拥有信誉
≠ 自动拥有合法性
```

因此现实系统必须由四层组成：

1. **密钥层**：负责认证；
2. **委托层**：负责授权；
3. **记录层**：负责信誉沉淀；
4. **治理层**：负责争议与合法性判断。

## 7. 恢复机制映射

## 7.1 世界观表达
失落之钥、主权恢复、社会见证、重建星球身份。

## 7.2 现实对应
- 加密备份；
- 助记词；
- Shamir secret sharing；
- recovery committee；
- time-locked recovery workflow；
- public recovery announcement。

## 7.3 恢复为何不能只是技术操作
如果恢复只是导入备份，那么：
- 被盗恢复无法与盗窃者区分；
- 双重恢复会制造平行合法性；
- 市场与治理无法知道当前主体是否稳定。

因此恢复还必须包括：
- 公共标记；
- 冷静期；
- Witness 确认；
- 高风险权限观察期。

---

## 8. 继承机制映射

## 8.1 世界观表达
领主死亡或失联后，主权延续给谁。

## 8.2 现实对应
- pre-registered succession policy；
- threshold witness confirmation；
- dispute arbitration；
- inheritance execution record。

## 8.3 关键边界
密码学无法独立回答继承问题。

它只能证明：
- 某份遗嘱或声明确实由过去的身份发出；
- 某人持有特定恢复材料；
- 某个委员会达到阈值签署。

真正回答继承的，是**制度顺序与争议裁决**。

---

## 9. 被盗与争议状态映射

## 9.1 世界观表达
盗印星球、争议星球、封存星球、失落星球。

## 9.2 现实对应
公共身份状态标记：

```json
{
  "identity_state": "active|suspected_compromise|contested|sealed|lost|archived",
  "public_flags": [
    "high_risk_market_actions_disabled",
    "governance_vote_frozen"
  ]
}
```

## 9.3 边界解释
这些状态标记不是本地主权剥夺，而是公共系统对风险的制度回应。

共同体可以：
- 暂停市场高风险操作；
- 冻结治理权；
- 标记争议；
- 要求重新验证。

共同体不应：
- 擅自抹除本地身份历史；
- 擅自接管本地数据；
- 直接判定星球“不存在”。

## 10. 与其他系统的现实接口

## 10.1 与连接系统
身份系统输出：
- 长期身份锚；
- Runtime 握手密钥；
- 争议/被盗状态标记。

连接系统据此决定：
- 是否接受高敏感建交；
- 是否要求额外确认；
- 是否允许建立高权限服务会话。

## 10.2 与治理系统
身份系统输出：
- 谁有资格提名治理代表；
- 当前是否处于 contested 状态；
- 恢复/继承是否完成。

治理系统据此决定：
- 票权是否冻结；
- 提案资格是否暂停；
- 继承体是否被共同体承认。

## 10.3 与经济系统
身份系统输出：
- 授权链；
- 代理范围；
- 风险状态。

经济系统据此决定：
- 托管是否释放；
- 高额订单是否要求额外验证；
- 担保是否需重建。

## 10.4 与声誉系统
身份系统输出：
- 身份连续性；
- 恢复/分裂事件记录；
- 委托责任链。

声誉系统据此决定：
- 哪些信誉维度可以继承；
- 哪些维度必须衰减重建；
- 分裂体是否共享旧信誉。

## 11. 一张总图：现实系统分层

```text
第 1 层：密码学层
- Ed25519 / X25519 / 哈希标识 / 多签

第 2 层：授权层
- Delegation Document / Capability Scope / Runtime Policy

第 3 层：公共记录层
- Reputation Ledger / Rotation Record / Recovery Notice / Dispute Record

第 4 层：制度层
- Succession Rule / Arbitration / Governance Recognition / Sanction Boundary
```

这四层共同构成世界观中的“身份主权体系”。
