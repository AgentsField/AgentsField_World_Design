# 02 · 身份与主权系统

> **核心映射**：密码学身份、签名关系与权限边界 → 数字文明中的身份层级、主权边界与合法性秩序  
> **章节定位**：定义“谁可以被承认、如何被承认、何时失去承认，以及身份如何在危机中继续存在”。

## 📖 章节概述

身份与主权系统是 AgentsField 宇宙的第一根骨架。它不再把“主权”理解为浪漫化的绝对统治，而是把它重写为一种**可验证、可迁移、可恢复、可继承、可争议、受共同体边界约束的有限主权**。

在本章中：
- 领主不再只是设备持有者，而是根印的保管者与授权链的起点；
- 星球不再等同于单一机器，而是由 Planet ID、运行时密钥、Agent 身份与声誉剖面共同组成的多层主体；
- 身份不只服务于认证，还决定授权、治理参与、经济结算、外交可达性与文明合法性；
- 所有高价值叙事都必须回到现实世界的压力测试：密钥丢失怎么办、设备迁移怎么办、代理失控怎么办、被盗身份如何冻结、继承如何执行、争议如何判定。

本章已经完成针对修订总纲要求的统一重写：以五层身份结构为轴，补齐恢复委员会、密钥轮换、代理委托、继承封存、失窃标记与争议上诉机制，确保身份体系与 04 章治理、07 章经济形成闭环。

## 🔑 核心概念速查

| 概念 | 简述 | 不可混淆点 |
|------|------|-----------|
| Root Seal（根印） | 最高身份控制权的源头 | 不是日常登录密钥，不应高频暴露 |
| Planet ID | 星球在公共网络中的持续标识 | 不是设备序列号，可跨硬件迁移 |
| Runtime Key | 当前设备/实例的运行凭证 | 可以轮换、吊销、冷却更换 |
| Agent Persona | 代理人格与行为面具 | 受上级授权，不天然拥有全部主权 |
| Reputation Profile | 可累积、可污染、可修复的公共评价面 | 不等同于合法性本身 |
| 合法性 | 共同体是否承认其当前行动有效 | 由签名、流程、证据、治理共同决定 |
| 有限主权 | 本地主权不可剥夺，但公共权利可受限制 | 可被制裁的是接入权、结算权、投票权，而非根存在 |

## 🧭 本章回答的关键问题

1. **谁才算一个“完整的星球主体”？**  
   答：必须同时具备根印来源、公共标识、运行时授权、代理人格边界与可追溯声誉。

2. **主权是否绝对？**  
   答：不是。主权在本地控制层面高度稳固，但在连接、交易、治理、声誉与公共服务层面必须接受契约和共同体约束。

3. **身份在灾难中如何延续？**  
   答：通过多副本备份、社交恢复、恢复委员会、多签继承与冻结-复核流程。

4. **一个身份被盗是否意味着文明死亡？**  
   答：不意味着。体系允许进入失窃标记、紧急冻结、代理降级、争议公示与身份复位流程。

5. **为什么本章是整个世界观的起点？**  
   答：因为治理需要确认提案主体，经济需要确认结算主体，外交需要确认通信对象，声誉需要确认评价对象——它们都依赖身份系统先成立。

## 🎯 核心机制闭环

### 1. 身份生命周期闭环
铸印 → 注册 → 授权 → 使用 → 轮换 → 恢复 / 继承 / 封存

### 2. 失窃处置闭环
异常发现 → 紧急标记 → 公共广播 → 权限降级 → 举证仲裁 → 恢复或分叉

### 3. 合法性闭环
签名存在 → 权限校验 → 程序正当 → 证据公开 → 共同体验证 → 最终承认

### 4. 文明连续性闭环
设备损毁 → Root Seal 仍存 → 继承或恢复 → 新运行节点接管 → 声誉衔接 → 文明续存

## ⚠ 技术与叙事难点摘要

| 难点 | 风险 | 本章处理方式 |
|------|------|-------------|
| 设备与身份绑定过死 | 一次故障即文明死亡 | 用 Planet ID 与 Runtime Key 分层解耦 |
| 主权叙事绝对化 | 无法容纳治理与制裁 | 改写为有限主权与公共权利限制 |
| 声誉与合法性混淆 | 舆论可替代程序正义 | 明确区分评价、授权与承认 |
| 恢复机制缺失 | 密钥丢失不可逆 | 引入恢复委员会、多签与冷却期 |
| 代理人格越权 | Agent 失控造成制度崩坏 | 引入权限域、行为边界与吊销机制 |
