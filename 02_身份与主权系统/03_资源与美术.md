## 🎨 视觉总原则

身份与主权系统的美术目标不是“做出一个漂亮的账号系统”，而是让玩家/读者一眼看出：
- 这是一个由印章、授权链与信誉痕迹构成的文明；
- 每个身份都拥有层级感、历史感与法律重量；
- 每一次签署、冻结、恢复、继承都应像政治仪式，而不是普通弹窗。

**风格关键词**：深空仪式感、密码学圣徽、冷金属、光学刻印、可信但不温吞、庄严但可交互。  
**主视觉气质**：神圣印章 × 高可信系统面板 × 悬浮式星图证书。

## 🖥 关键界面设计说明

### 1. 身份总览面板
应展示以下模块：
- 当前主身份名与 Planet ID；
- 根印是否离线保管；
- 当前运行时密钥数量与风险等级；
- 已委托的 Agent Persona 列表；
- 声誉剖面摘要；
- 是否存在争议、冻结、继承、恢复流程。

**交互要求**：
- 高权限操作需二次确认，并伴随“签署仪式”动效；
- 风险状态必须以颜色、裂痕、警示边框三层共同表达；
- 不允许把冻结、被盗、恢复中等高风险状态做成弱提示。

### 2. 铸印仪式页
流程动效建议：
1. 熵源汇聚为星尘；
2. 星尘压缩形成金属印核；
3. 印核刻出根纹；
4. Planet ID 自印核中投影而出；
5. 用户确认保管方式（冷存储 / 助记词 / 分片恢复）；
6. 完成时出现庄严、短促、明确的合法性提示。

### 3. 恢复与继承页
必须突出“这是一次政治与法律动作”，而不是普通找回密码：
- 展示恢复委员会席位；
- 标示已提交证明、待确认见证、冷却期倒计时；
- 展示恢复后哪些 Runtime Key 将被吊销；
- 用双栏并列“旧身份残响”与“新身份承继”。

## 🎞 动效设计

| 场景 | 动效描述 | 时长建议 | 反馈目标 |

| 继承生效 | 旧印章沉入暗处，新印章自其上升起 | 1.5s | 表现延续，而非替代 |

## 📚 视觉叙事母题

1. **刻印**：一切身份都必须留下痕迹。  
2. **裂痕**：一切失序都必须被看见。  
3. **缝合**：恢复不是清空，而是带着伤口继续存在。  
4. **投影**：公共身份永远只是根印的投影，不是根印本身。  
5. **层叠**：身份不是单张证件，而是一层一层叠出的文明肖像。
