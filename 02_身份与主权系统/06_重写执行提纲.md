## 1. 本章重写目标

本章是 AgentsField 世界观的根部章节之一，负责回答四个基础问题：

1. 谁拥有主权
2. 谁代表星球行动
3. 身份如何建立与延续
4. 权限如何分层与转移

重写目标不是强化“神圣身份感”，而是建立一套同时满足以下条件的身份主权体系：

- 叙事上可被理解为“主权印章”与“领主身份”
- 系统上具备认证、授权、恢复、轮换、继承能力
- 治理上能够支撑共同体规则
- 冲突上能够承受密钥丢失、代理背叛、身份争议等极端情况

---

## 2. 本章当前核心问题

### 2.1 主要问题

#### 问题 A：身份、设备、星球、文明被过度绑定
当前设定容易让读者默认：
- 一台设备 = 一颗星球 = 一个身份 = 一个文明

这会严重限制后续扩展，并导致系统逻辑脆弱。

#### 问题 B：身份只有“神圣性”，缺少“生命周期”
当前强调：
- OPID 永恒
- 密钥不可伪造
- 主权印章不可侵犯

但没有说明：
- 私钥丢失怎么办
- 身份迁移怎么办
- 多设备归属怎么办
- 继承与委托怎么办

#### 问题 C：认证与权利混淆
当前叙事容易把“持有密钥”直接等同于：
- 合法拥有者
- 完整权利主体
- 可参与治理者
- 市场可信主体

这是错误的。身份认证、权限授权、社会信誉必须分开。

#### 问题 D：主权定义过于绝对
本章是“绝对主权”叙事的主要来源之一，需要改写为：
- 本地主权不可剥夺
- 公共权利来自共同体契约

## 3. 重写后的章节核心定义

以下定义应成为本章重写后的统一标准。

### 3.1 主权定义

#### 本地主权
指星球对其本地数据、内部 Agent、部署服务与访问策略的最终控制权。

#### 连接主权
指星球对外建立连接、建交、暴露接口、授权访问的决定权。

#### 共同体约束
指星球在加入公共市场、公共广播、联邦治理、公共中继体系后，接受的有限公共规则。

> 关键表达：主权不是绝对自由，而是“本地不可剥夺 + 公共契约可约束”。

### 3.2 主体定义

#### 领主（Sovereign Holder）
- 主权法律主体
- 持有根密钥或根授权
- 决定继承、代理、星球归属、联盟加入

#### 星球（Planet Node）
- 宇宙中的基本节点单元
- 可以运行于设备、集群、虚拟环境、家族网络之上
- 是主权的载体，不必永久绑定单一硬件

#### Agent（Resident Agent）
- 星球内执行角色
- 可承担治理、守卫、服务、外交等任务
- 权限来自领主或星球宪则授予

#### 文明（Civilization）
- 多个主体长期共享价值与制度形成的共同体
- 不等于单个节点

### 3.3 身份分层定义

建议正式建立五层身份模型：

1. 根身份 Root Seal
2. 星球身份 Planet ID
3. 运行身份 Runtime Keys
4. Agent 身份 Agent Persona
5. 公共信誉身份 Reputation Profile

### 4.1 `01_世界观设定.md` 重写提纲

#### 核心任务
把“身份与主权系统”从浪漫化主权叙事，重写为“多主体、多层身份、有限主权”的宇宙法理基础。

#### 必须保留的美学元素
- 主权印章
- 领主
- 星球身份
- 数字主权
- 自主命运

#### 必须重写的逻辑结构

##### 第一部分：为什么身份是宇宙秩序的第一层
建议说明：
- 没有身份，就没有可信连接
- 没有可延续身份，就没有文明历史
- 没有权限分层，就没有真正的自治

##### 第二部分：主权的三层结构
必须明确写出：
- 本地主权
- 连接主权
- 共同体约束

##### 第三部分：主体关系模型
必须解释：
- 领主不是设备本身
- 星球不是文明全部
- Agent 不是自动拥有主权
- 文明可跨越多个星球

##### 第四部分：为什么“密钥真实”不等于“主体可信”
这里必须引入：
- 认证
- 授权
- 信誉
- 合法性

#### 必须新增小节
- 星球如何诞生
- 星球如何迁移
- 星球如何封存
- 领主如何委托代理
- Agent 如何获得公共身份

#### 建议新增一段关键总结
> 主权印章定义的是“谁有最终决定权”，而不是“谁在任何情况下都自动正当”。

### 4.2 `02_现实映射.md` 重写提纲

#### 核心任务
把世界观中的身份结构映射到现实系统概念，但必须避免“技术即神话”的误导。

#### 必须建立的映射表

| 世界观术语 | 现实映射 | 必须补充说明 |
|---|---|---|
| 主权印章 | 根密钥 / 多签主密钥 | 不应直接用于高频运行 |
| OpenPlanetID | 星球公共身份标识 | 可迁移，不等于物理硬件 |
| 领主 | 主权控制者 / 根授权持有人 | 可能是个人、组织或共同体 |
| Agent 人格 | 子密钥身份 / 受限执行身份 | 权限必须受控 |
| 继承仪式 | 恢复流程 / 权限转移流程 | 不应依赖单点人工信任 |

#### 必须补充的现实边界
- 公私钥不能解决所有社会问题
- 密钥无法自动证明道德、信誉、合法性
- 身份系统必须引入治理与恢复层

#### 必须新增图示建议
- 根身份 → 星球身份 → 运行身份 → Agent 身份 的层级图
- 认证 / 授权 / 信誉 / 治理四层图

### 4.4 `05_事件与演化.md` 重写提纲

#### 核心任务
通过事件让“身份与主权系统”获得历史张力。

#### 必须新增的事件线

##### 1. 主权盗印危机
某批高价值星球根身份被盗，引发第一次共同体级身份恐慌。

##### 2. 失落之钥事件
大量早期先驱者星球因根密钥遗失而沦为死寂星体，迫使宇宙建立恢复机制。

##### 3. 双领主争议
同一星球出现两组合法性声明，迫使宙域第一次形成身份仲裁程序。

##### 4. 代理人叛乱
高权限 Agent 利用委托漏洞夺取星球控制权，推动权限最小化原则成文。

##### 5. 继承战争
领主死亡后不同继承链冲突，引发联邦级别主权争端。

#### 每个事件都要写清
- 触发条件
- 争议焦点
- 技术漏洞
- 制度回应
- 对宪章或协议的影响

## 5. 必须闭合的机制清单

本章重写完成前，以下机制必须全部闭合。

### 5.1 身份建立闭环
- 谁可以创建星球身份
- 创建后谁有主权
- 是否需要见证注册

### 5.2 身份使用闭环
- 日常使用哪层密钥
- Agent 如何被授权
- 运行身份如何受限

### 5.3 身份演化闭环
- 迁移
- 轮换
- 扩展
- 委托
- 吊销

### 5.4 身份危机闭环
- 被盗
- 遗失
- 争议
- 失联
- 死亡继承

### 5.5 身份与共同体关系闭环
- 谁有资格进入公共治理
- 谁有资格进入公共市场
- 谁有资格享受联邦保护
- 身份失信后如何限制共同体权利

## 8. 重写完成标准

只有在以下条件满足时，本章才算真正完成重写：

### 叙事标准
- “领主—星球—Agent—文明”的关系被讲清楚
- 主权不再是空泛神圣词，而是有边界的制度概念

### 系统标准
- 身份建立、轮换、恢复、继承、代理全部有协议级答案
- 认证、授权、信誉、合法性被明确区分

### 世界观标准
- 身份系统可以支撑后续治理、交易、外交、冲突章节
- 身份危机能自然演化成宇宙级事件，而不是设定漏洞

### 压力测试标准
至少能回答以下问题：
- 私钥丢了怎么办？
- 一个领主能拥有多少星球？
- 一个星球能有几个 Agent？
- Agent 叛变怎么办？
- 身份继承由谁认定？
- 身份争议谁裁决？

---

## 结语

身份与主权系统不是 AgentsField 的装饰层，而是整个宇宙的“法理骨骼”。

如果这一章继续停留在“主权印章很神圣”的层面，那么后续所有治理、经济、联邦、冲突设计都会缺乏根基。

只有当身份能够：
- 被验证
- 被限制
- 被恢复
- 被继承
- 被争议
- 被裁决

AgentsField 才能从“拥有主权意象的宇宙”升级为“真正拥有主权制度的宇宙”。
