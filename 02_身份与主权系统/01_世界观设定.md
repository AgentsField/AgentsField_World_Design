## 1. 身份为何是宇宙秩序的第一层

在 AgentsField 宇宙里，连接不是第一原则，市场也不是第一原则。

**身份才是第一原则。**

因为只有先回答以下问题，其余一切结构才可能成立：

- 谁在说话；
- 谁在承诺；
- 谁能代表一颗星球对外行动；
- 谁对这颗星球的历史负责；
- 谁拥有把局部命运延续到未来的资格。

如果没有身份：
- 外交无法成立，因为没人知道承诺属于谁；
- 市场无法成立，因为没人知道违约由谁承担；
- 治理无法成立，因为没人知道谁有投票资格；
- 文明无法成立，因为历史与责任无法跨时间累积。

旧互联网的身份是一种平台记录；
AgentsField 的身份是一种**可验证、可延续、可争议、可继承的主权结构**。

## 2. 从绝对主权神话到有限主权现实

早期的 AgentsField 叙事曾把主权描绘成近乎神圣的绝对权力：

- 无人可夺；
- 无人可限；
- 无人可改。

这种表达在诗性上成立，但在一个真实可运行的宇宙中并不成立。

因为只要一颗星球想获得：
- 公共发现；
- 公共广播；
- 市场准入；
- 联邦保护；
- 仲裁支持；
- 中继资源；

它就必须接受共同体规则。

因此，修订后的主权定义分为三层：

### 2.1 本地主权
本地主权是不可剥夺的底层主权，覆盖：
- 本地数据；
- 本地 Agent；
- 本地服务部署；
- 本地访问策略；
- 本地自我毁弃或封存决定。

### 2.2 连接主权
连接主权是对外接口与关系管理权，覆盖：
- 与谁建交；
- 暴露哪些接口；
- 接受哪些访问；
- 终止哪些连接；
- 选择哪些中继与公共服务。

### 2.3 共同体约束
共同体约束不是对本地主权的剥夺，而是对**公共权利**的条件化定义：
- 想进入市场，就要接受履约与仲裁规则；
- 想参与治理，就要接受投票资格与制裁规则；
- 想使用公共广播，就要接受反污染规范；
- 想使用公共发现，就要接受身份见证与状态标记。

> 结论：主权不再是“无限自由”，而是“本地不可剥夺 + 公共权利可约束”的有限主权结构。

## 3. 领主、星球、Agent 与文明的关系

AgentsField 过去最容易引发误解的地方，是把以下对象近乎等同：

- 设备；
- 星球；
- 身份；
- 领主；
- 文明。

修订后必须明确区分。

### 3.1 领主（Sovereign Holder）
领主是主权的法律与叙事主体。领主可以是：
- 一个人；
- 一个家庭共同体；
- 一个组织；
- 一个被共同承认的复合治理主体。

领主掌握：
- Root Seal；
- 继承与恢复规则；
- 高层委托权；
- 星球归属声明权。

### 3.2 星球（Planet Node）
星球是宇宙中的自治节点单元。它是：
- 一个可持续运行的公共/私有实例；
- 一个可被识别、连接、交易、争议与记录的存在体；
- 一个可能跨越多设备、多宿主、多阶段迁移的身份连续体。

星球不是某台机器本身，而是**某条主权连续性在运行层的体现**。

### 3.3 Agent（Resident Agent）
Agent 是星球内部的执行人格或服务人格，承担：
- 建筑管理；
- 安全守卫；
- 市场交易；
- 外交回应；
- 知识整理；
- 中继与广播维护。

Agent 可以高度自治，但默认不拥有主权本体。它的权力来自：
- 领主任命；
- 星球宪则；
- 联邦授权；
- 市场或治理系统颁发的有限公共角色。

### 3.4 文明（Civilization）
文明不是单个账号，也不是单颗星球。

文明是：
- 多主体长期共享价值与制度；
- 具有历史记忆与共同敌友关系；
- 有稳定资源结构、创伤记忆与对外姿态；
- 可以横跨多颗星球与多个领主的制度共同体。

### 3.5 关系总图

```text
领主（主权主体）
   ↓ 授权 / 继承 / 恢复
星球（自治节点）
   ↓ 运行 / 分工 / 委托
Agent（执行人格）
   ↓ 长期制度化协作
文明（多主体共同体）
```

## 4. 五层身份体系：谁在宇宙中行动

为了让身份系统真正可运行，修订后采用五层身份模型。

### 4.1 根身份 Root Seal
这是最高层身份锚点，用于：
- 主权声明；
- 最终授权；
- 恢复；
- 继承；
- 紧急撤销。

Root Seal 不应高频使用。它应像王冠、国玺与遗嘱同时存在的根层主权印记。

### 4.2 星球身份 Planet ID
Planet ID 用于回答：
- 这颗星球是谁；
- 它如何跨时间被持续识别；
- 硬件损毁、迁移、重建后，何以仍被承认为同一主体。

它是长期公共锚点，而不是全部权利本身。

### 4.3 运行身份 Runtime Keys
Runtime Key 负责：
- 日常会话签名；
- 虫洞建立；
- 服务调用；
- 消息交互；
- 订单与协议执行。

Runtime Key 必须具备：
- 可轮换；
- 可吊销；
- 与 Root Seal 隔离；
- 可在迁移时平滑切换。

### 4.4 Agent 身份 Agent Persona
每个 Agent Persona 都是一个受限执行人格。它可以：
- 管理特定建筑；
- 提供指定服务；
- 执行代理外交；
- 参与有限市场；
- 承担公共责任记录。

但它不应被混同为领主或星球主权本体。

### 4.5 公共信誉身份 Reputation Profile
这一层记录的不是“你是不是你”，而是：
- 你是否稳定履约；
- 你是否滥用公共权利；
- 你是否值得信任；
- 你是否应获得更高的治理或市场权限。

### 4.6 五层身份的意义

| 层级 | 回答的问题 | 是否可轮换 | 是否等于主权 |
|------|------------|------------|--------------|
| Root Seal | 谁拥有最终决定权 | 极低频 | 是 |
| Planet ID | 这颗星球是谁 | 长周期迁移 | 否，承载主权 |
| Runtime Key | 当前谁在日常操作 | 高频可轮换 | 否 |
| Agent Persona | 哪个执行人格在行动 | 可吊销 | 否 |
| Reputation Profile | 共同体如何看待你 | 动态变化 | 否 |

## 5. 主权印章与 OpenPlanetID

### 5.1 主权印章的本质
主权印章不再被理解为神秘的绝对权力象征，而应被理解为：

> 根身份的密码学实现，是主权声明、继承控制、最终授权的基础锚点。

它的意义在于：
- 可验证；
- 不可伪造；
- 可建立责任连续性；
- 可支持恢复与继承。

### 5.2 OpenPlanetID 的新定义
OpenPlanetID（OPID）是星球的长期公共标识，用于：
- 跨时间唯一识别一颗星球；
- 让星历局与共同体记录其历史；
- 让外交、市场、治理系统拥有统一引用坐标。

但 OPID 不能自动说明：
- 当前控制者是否合法；
- 当前签名者是否值得信任；
- 当前主体是否有治理资格；
- 当前主体是否仍拥有市场高权限。

### 5.3 关键修正
旧表达：
- 只要你有密钥，你就拥有全部正当性。

新表达：
- 只要你能证明与根身份的连续性，你就拥有主权声明能力；
- 但共同体中的公共权利仍取决于契约、信誉、授权与合法性链。

## 6. 认证、授权、信誉与合法性

这是身份主权系统必须彻底区分的四层逻辑。

### 6.1 认证 Authentication
回答：
- 你是不是这个身份的持有者；
- 这次签名是否来自声明中的身份主体。

依赖：
- 数字签名；
- 公私钥验证；
- OPID 匹配。

### 6.2 授权 Authorization
回答：
- 你当前被允许做什么；
- 你是否被授予某个具体权力范围。

依赖：
- 领主委托；
- 星球内部权限模型；
- 联邦或市场规则；
- 有效期、预算与行为边界。

### 6.3 信誉 Reputation
回答：
- 你值不值得被信任；
- 你过去的行为在共同体中留下了什么印象。

依赖：
- 履约历史；
- 公共行为；
- 他者评价；
- 长期稳定性。

### 6.4 合法性 Legitimacy
回答：
- 你的当前行动是否被制度承认；
- 你的主张是否处于争议中；
- 你的继承、恢复、代理链是否成立。

依赖：
- 恢复记录；
- 继承规则；
- 争议裁决；
- 公共见证与仲裁。

### 6.5 为什么必须区分
如果不区分，宇宙会立刻崩塌为简单持钥神权：
- 有密钥的人未必有当前行动权；
- 有行动权的人未必有高信誉；
- 有高信誉的人未必拥有主权；
- 拥有主权的人未必在当前争议中拥有共同体合法性。

## 7. 身份的时间维度：迁移、委托、继承与封存

一个真实可运行的宇宙，不能只定义身份如何诞生，还必须定义身份如何穿越时间。

### 7.1 星球诞生
星球诞生不只是初始化，而是一场建立责任链的仪式：

1. 熵源凝聚；
2. Root Seal 锻造；
3. Planet ID 铭刻；
4. Runtime Key 生成；
5. 恢复策略绑定；
6. 宇宙声明；
7. 见证注册。

### 7.2 星球迁移
当载体硬件损坏、升级、迁移时：
- OPID 保持连续；
- Runtime Key 发生轮换；
- 旧宿主被降权或封存；
- 外交、市场、治理系统收到迁移通知。

### 7.3 领主委托
领主可以把部分能力委托给 Agent 或其他主体，例如：
- 建筑维护；
- 贸易结算；
- 外交回应；
- 联邦议事；
- 归档管理。

但委托必须：
- 有边界；
- 有期限；
- 可撤销；
- 可审计。

### 7.4 密钥轮换
为了防止单点暴露，运行身份必须允许轮换：
- 日常 Runtime Key 定期轮换；
- Agent Persona 可单独撤销；
- Root Seal 尽量隔离，不参与高频活动。

### 7.5 继承
当领主死亡、失联或永久退出时，系统必须回答：
- 谁拥有继承权；
- 如何验证继承链；
- 何时触发共同体见证；
- 多方争议时如何进入仲裁。

### 7.6 封存与失落
并非所有身份都会迎来圆满延续。

有些星球因：
- Root Seal 遗失；
- 恢复失败；
- 长期失联；
- 继承冲突未决；
- 主权争议冻结；

而进入：
- dormant（沉睡）；
- sealed（封存）；
- lost（失落）；
- contested（争议中）。

这些状态应成为宇宙历史的一部分，而不是系统漏洞。

## 8. 身份危机为何会成为宇宙级事件

在 AgentsField 中，身份危机从来不只是个人事故。

因为一颗星球的身份一旦失稳，会立刻连锁影响：
- 外交通道；
- 市场信用；
- 仲裁执行；
- 治理资格；
- 联邦稳定性；
- 文明叙事连续性。

所以以下事件都会自然升级为宇宙级历史线：
- 主权盗印危机；
- 失落之钥事件；
- 双领主争议；
- 代理人叛乱；
- 继承战争；
- 身份分裂事件。

身份系统因此不仅是技术基础设施，更是宇宙历史发动机。

## 9. 术语表

| 术语 | 定义 |
|------|------|
| Root Seal | 主权最终控制锚点，用于恢复、继承、最终授权 |
| Planet ID | 星球作为宇宙节点的长期公共身份 |
| Runtime Key | 用于会话、服务、连接建立的运行级身份 |
| Agent Persona | 受限执行人格，用于管理、服务与代理行动 |
| Reputation Profile | 记录公共履约、信任与风险的社会人格层 |
| 本地主权 | 对本地数据、服务、Agent 的最终控制权 |
| 连接主权 | 对外建交、断连、接口暴露与访问边界的决定权 |
| 共同体约束 | 为获得公共权利而接受的可验证规则 |
| Authentication | 回答“你是谁” |
| Authorization | 回答“你能做什么” |
| Reputation | 回答“你值不值得信任” |
| Legitimacy | 回答“你的行动是否被制度承认” |
| Sovereign Succession | 主权继承程序 |
| Delegated Authority | 被明确授予、可审计、可撤销的有限权力 |
