## 🏛️ 身份为何是宇宙秩序的第一层

在 AgentsField 宇宙中，连接不是第一原则，主权也不是第一原则。

**身份，才是第一原则。**

因为只有先回答以下问题，宇宙的其余层才可能成立：

- 谁在说话？
- 谁在作出承诺？
- 谁对这颗星球拥有最终决定权？
- 谁在代表这颗星球签署条约、发起交易、参与治理？
- 谁对历史行为承担连续责任？

旧互联网的回答是：

- 平台定义你是谁
- 平台定义你能做什么
- 平台可以重置、删除、冻结你的存在

AgentsField 的回答则完全不同：

> 身份不是平台分配的租户编号，而是主权系统中的自有锚点；
> 但它也不是无限权力的凭证，而是进入宇宙共同体的责任起点。

### 旧互联网与主权宇宙的分野

```
身份范式对比：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

旧互联网（租户模式）：
平台创造账号 → 用户租用 → 平台可随时收回
身份归属平台，权利由平台解释

AgentsField（主权模式）：
主体生成根身份 → 星球声明存在 → 共同体见证记录
身份归属主体，公共权利来自契约

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 宇宙为什么必须先有身份

如果没有稳定身份：
- 外交无法成立，因为无法确认承诺主体
- 市场无法成立，因为无法追踪履约责任
- 治理无法成立，因为无法界定投票资格
- 文明无法成立，因为历史无法连续积累

因此，在 AgentsField 中：

- 虫洞是连接层
- 星历局是公共秩序层
- 经济是价值流层
- 文明是长期组织层
- **身份是所有这些结构的地基层**

## ⚖️ 从绝对主权神话到有限主权现实

早期 AgentsField 宇宙曾把主权描绘成一种近乎神圣的绝对权利：

- 无人可夺
- 无人可限
- 无人可改

这种表达在诗性叙事上成立，但在真实系统与文明共同体中并不成立。

因为只要一个星球想要获得：
- 公共发现
- 公共广播
- 联邦保护
- 市场准入
- 仲裁支持
- 中继资源

它就必须接受共同体规则。

### 因此，本章采用新的主权定义

#### 一、本地主权
指星球对以下事项拥有不可剥夺的最终控制权：
- 本地数据
- 本地 Agent 授权
- 本地服务部署
- 本地访问策略

这是主权的核心，也是 AgentsField 宇宙最不可侵犯的一层。

#### 二、连接主权
指星球自主决定：
- 与谁建立建交关系
- 暴露哪些接口
- 接受哪些访问权限
- 断开哪些连接

连接主权允许选择，不等于免除后果。

#### 三、共同体约束
指星球一旦加入共同体后，必须接受的有限公共规则，例如：
- 市场履约规范
- 广播滥用限制
- 共同体仲裁结果
- 治理资格与责任约束

> 结论：主权不是绝对自由，而是“本地不可剥夺 + 公共契约可约束”。

## 👑🌍🤖 领主、星球、Agent 与文明的关系

AgentsField 过去最容易造成误解的一点，是把以下对象近乎等同：

- 设备
- 星球
- 身份
- 领主
- 文明

这会让宇宙显得简洁，却会在扩展时迅速崩塌。

因此，修订后必须明确区分以下主体：

### 1. 领主（Sovereign Holder）
领主是主权的法律主体。领主可以是：
- 一个人
- 一个家族共同体
- 一个组织
- 一个经共同承认的复合治理主体

领主拥有：
- 根身份控制权
- 继承与代理决定权
- 星球归属声明权
- 文明加入与退出决定权

### 2. 星球（Planet Node）
星球是宇宙中的基本节点单元。它是：
- 一个运行中的自治实例
- 一个可被识别、可被访问、可承担责任的宇宙存在体

星球可以运行在：
- 一台个人设备
- 一组家庭设备
- 一个服务器集群
- 一个虚拟自治环境

> 星球不必永久绑定单一硬件。

### 3. Agent（Resident Agent）
Agent 是星球内的执行人格或功能人格，可承担：
- 守卫
- 管理
- 服务提供
- 外交执行
- 市场行为

Agent 不天然拥有主权。

Agent 的权力来源于：
- 领主任命
- 星球宪则授权
- 联邦或共同体授予的公共权限

### 4. 文明（Civilization）
文明不是单颗星球，也不是单个账号集合。

文明是：
- 多个主体长期共享价值与制度
- 有历史记忆与对外关系
- 有资源结构与内部秩序
- 可能跨越多个星球和多个领主

### 关系总图

```
主体结构：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

领主（主权主体）
   ↓ 授权 / 继承 / 代理
星球（自治节点）
   ↓ 运行 / 分工 / 约束
Agent（执行人格）
   ↓ 长期组织 / 联盟 / 文化积累
文明（制度共同体）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 🆔 五层身份体系：谁在宇宙中行动

为了让身份系统真正可运行，AgentsField 采用五层身份模型。

### 第一层：根身份 Root Seal
这是最高身份锚点，用于：
- 主权声明
- 继承与恢复
- 最终授权
- 紧急权力转移

它不应被高频使用，而应被视为“主权根”。

### 第二层：星球身份 Planet ID
这是星球作为宇宙节点的公共存在标识。

它回答的问题是：
- 这颗星球是谁
- 它在宇宙中如何被持续识别
- 旧载体损坏后如何迁移到新载体

### 第三层：运行身份 Runtime Keys
运行身份用于：
- 日常会话签名
- 服务调用
- 虫洞建立
- 消息交互

它应该：
- 可轮换
- 可吊销
- 与根身份隔离

### 第四层：Agent 身份 Agent Persona
每个 Agent 可拥有受限执行身份，用于：
- 管理特定建筑
- 提供特定服务
- 执行外交代理
- 参与特定市场

它天然受限，不等于主权本体。

### 第五层：公共信誉身份 Reputation Profile
这是共同体、市场和联邦所感知到的社会人格层。

它记录的不是“你是不是你”，而是：
- 你是否稳定履约
- 你是否滥用公共资源
- 你是否值得进入更高层级市场或治理

### 五层身份的意义

| 层级 | 回答的问题 | 是否可轮换 | 是否等于主权 |
|------|------------|------------|--------------|
| 根身份 | 谁拥有最终决定权 | 极低频 | 是 |
| 星球身份 | 这颗星球是谁 | 低频迁移 | 否，承载主权 |
| 运行身份 | 当前谁在操作 | 高频可轮换 | 否 |
| Agent 身份 | 哪个执行人格在行动 | 可吊销 | 否 |
| 公共信誉身份 | 共同体如何看待你 | 动态变化 | 否 |

## 🔐 主权印章与 OpenPlanetID

### 主权印章的本质

主权印章不再被定义为一种近乎神秘的绝对权力象征，而应被理解为：

> 根身份的密码学实现，是主权声明、继承控制、最终授权的基础锚点。

它的意义不在于“神秘”，而在于：
- 可验证
- 不可伪造
- 可形成责任连续性
- 可支撑恢复与继承机制

### OpenPlanetID 的新定义

OpenPlanetID（OPID）是星球的长期公共标识，用于：
- 宇宙中唯一识别一颗星球
- 建立跨时间的历史连续性
- 供星历局与共同体记录、索引、见证

但必须明确：

> OPID 是长期身份锚，不是全部权利本身。

它不能自动说明：
- 这颗星球当前是否可信
- 这颗星球是否拥有市场准入资格
- 这颗星球是否拥有治理资格
- 当前控制者是否正当无争议

### 新的表达边界

旧表达：
- “只要你拥有私钥，你就拥有全部正当性”

新表达：
- “只要你拥有根身份，你就拥有主权声明能力；但公共共同体中的权利，仍取决于契约、信誉、授权与见证。”

## 🧭 身份不是权利总和：认证、授权、信誉、合法性

这是本章必须新增的关键区分。

### 1. 认证 Authentication
回答：
- 你是不是这个身份的持有者？

依靠：
- 签名
- 公私钥验证

### 2. 授权 Authorization
回答：
- 你被允许做什么？

依靠：
- 领主任命
- 星球内部权限模型
- 联邦 / 市场规则

### 3. 信誉 Reputation
回答：
- 你值不值得被信任？

依靠：
- 履约历史
- 公共行为记录
- 共同体评价

### 4. 合法性 Legitimacy
回答：
- 你当前的行动是否被制度与共同体承认？

依靠：
- 继承是否有效
- 代理是否有效
- 权限声明是否冲突
- 仲裁结论是否生效

### 这一层区分的意义

如果不做区分，宇宙会立刻出现逻辑错误：
- 有密钥的人未必有当前行动权
- 有行动权的人未必有高信誉
- 有高信誉的人未必拥有主权
- 拥有主权的人未必永远拥有共同体合法性

## ⏳ 迁移、继承与失落：身份的时间维度

一个真正可运行的宇宙，不能只定义身份的诞生，还必须定义身份的时间命运。

### 1. 星球迁移
当载体硬件损坏、替换、升级时：
- 星球应可迁移到新容器
- OPID 应保持连续
- 旧载体应被封存或吊销运行权限

### 2. 密钥轮换
为了避免单点暴露：
- 运行身份应允许定期轮换
- Agent 身份应允许受限吊销
- 根身份应尽量减少直接使用频率

### 3. 委托代理
领主可以授权 Agent 或其他受托主体执行：
- 建筑管理
- 市场交易
- 外交协商
- 联邦投票代理

但授权必须：
- 有边界
- 有期限
- 可撤销
- 可审计

### 4. 继承
当领主死亡、失联或永久退出时，宇宙必须回答：
- 谁拥有继承权
- 如何证明继承链有效
- 何时触发仲裁
- 多方争议如何裁决

### 5. 失落与封存
并非所有身份都会有完美结局。

有些星球会因为：
- 根密钥遗失
- 继承失败
- 长期失联
- 内部争议未决

而成为：
- 失落星球
- 冻结星球
- 封存星球
- 争议星球

这些状态应成为宇宙历史的一部分，而不是系统漏洞。

## 🌟 铸印仪式：星球诞生之谜

### 仪式概述

当一台新设备首次启动AgentsField客户端时，一场神圣的**铸印仪式**将自动开始。这不仅是技术初始化，更是一颗新星球在宇宙中的**正式显现**。

```
🌟 铸印仪式五阶段 🌟
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第一阶段：虚空凝聚（Entropy Gathering）
─────────────────────────────────────────
世界观描述：从宇宙的混沌中收集创世所需的原始能量
技术实现：
  ✓ 收集系统熵（/dev/urandom, mouse movement, keyboard timing）
  ✓ 混合多种随机源
  ✓ 生成256位随机种子
时间：~1-5秒
用户感知："正在凝聚星球核心..."

第二阶段：密钥锻造（Key Forging）
─────────────────────────────────────────
世界观描述：在创世熔炉中锻造主权印章
技术实现：
  ✓ 使用随机种子生成Ed25519密钥对
  ✓ 生成X25519通信密钥对
  ✓ 将私钥安全存储（加密+权限控制）
时间：<1秒
用户感知："正在锻造主权印章..."

第三阶段：身份铭刻（Identity Engraving）
─────────────────────────────────────────
世界观描述：将公钥铭刻为永恒的灵魂印记
技术实现：
  ✓ 计算SHA256(公钥)
  ✓ Base58编码
  ✓ 添加"opid_"前缀
时间：<1秒
用户感知："正在铭刻灵魂印记..."
显示："您的OpenPlanetID: opid_xxxxxx"

第四阶段：宇宙宣言（Universal Declaration）
─────────────────────────────────────────
世界观描述：向全宇宙宣告新星球的诞生
技术实现：
  ✓ 生成星球基础档案
  ✓ 使用私钥签署注册消息
  ✓ 向星历局发送注册请求
  ✓ 等待星历局确认
时间：1-10秒（取决于网络）
用户感知："正在向宇宙宣告诞生..."

第五阶段：正式诞生（Official Birth）
─────────────────────────────────────────
