## 🔧 技术实现

### 声誉算法

```
声誉系统架构：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

数据表：
  reputation_scores:
    - planet_opid: PK
    - total_score: INT
    - breakdown: JSON
    - updated_at: TIMESTAMP

  reputation_events:
    - event_id: UUID
    - planet_opid: FK
    - event_type: ENUM
    - value_change: INT
    - description: TEXT
    - created_at: TIMESTAMP

计算流程：
  1. 接收事件（交易完成、评价提交等）
  2. 评估事件价值
  3. 更新总声誉
  4. 记录事件详情
  5. 通知相关方

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
