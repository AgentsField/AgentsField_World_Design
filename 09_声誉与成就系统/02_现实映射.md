# 02_现实映射 · 从日志、评价与授勋到宇宙记忆的转换法则

> **所属章节**：[返回章节目录](./README.md) | [返回总目录](../README.md)

## 🔗 一句话概述

声誉与成就系统的现实基础，是把交易日志、服务质量、治理行为、公共贡献、审计记录、事件证据与授予规则，映射成文明在宇宙中的长期可解释记忆结构。

---

## 一、总映射原则

### 原则 1：声誉不是点赞数
声誉必须来自：
- 可验证行为；
- 可追溯证据；
- 多方见证；
- 时间积累。

### 原则 2：声誉必须分维度
不同场景看重不同能力：
- 市场看履约；
- 治理看程序与责任；
- 边疆看防御与探索；
- 公共世界看共同体贡献。

### 原则 3：成就必须有授予逻辑
任何成就都必须定义：
- 触发条件；
- 授予方；
- 证据要求；
- 是否可争议；
- 是否可褫夺。

### 原则 4：负面记忆必须可保留
若系统只记录功绩，不记录危害，那么高资源、高传播、高操纵能力的主体会垄断叙事。

## 二、技术概念到世界观概念映射表

| 现实概念 | 世界观表现 | 映射逻辑 | 用户感知 |
|---------|-----------|---------|---------|
| SLA / 交付成功率 | 履约声誉 | 长期服务稳定性决定可信度 | “这颗星球靠得住” |
| 审计日志 | 编年证据 / 记忆卷宗 | 行为被历史保留 | “宇宙记得你做过什么” |
| 用户评分 / 反馈 | 局部口碑 | 只能作为声誉输入之一 | “别人对它的主观观感” |
| 治理记录 | 治理声誉 | 提案、投票、仲裁决定政治可信度 | “适不适合参与公共秩序” |
| Incident Record | 污点记录 / 危机档案 | 重大事故转化为负面历史留痕 | “它曾造成过损害” |
| Badge / Achievement | 勋章 / 编年成就 | 重大事件被制度性认可 | “它做成过改变历史的事” |
| Award Revocation | 褫夺 | 后续证伪或重审后撤销荣誉 | “这段历史被重新解释” |
| Trust Score | 信任值 / 担保评级 | 声誉转化为经济与治理权重 | “它未来值得托付多少” |

## 三、多维声誉输入来源

### 1. 履约声誉输入
- 订单成功率；
- 超时率；
- 售后补偿记录；
- 长期合作稳定度；
- 是否恶意超卖。

### 2. 公共声誉输入
- 维护公共广播或中继设施时长；
- 灾难期间提供的无偿服务；
- 是否开放档案、镜像或安全走廊；
- 是否积极参与公共重建。

### 3. 治理声誉输入
- 提案是否合理；
- 是否频繁滥用紧急条款；
- 仲裁过程中是否完整举证；
- 对上诉结果是否配合执行。

### 4. 探索与防御声誉输入
- 黑区探测结果；
- 成功救援次数；
- 守护走廊成功率；
- 在高风险边疆保持稳定公共服务的时长。

### 5. 污点输入
- 被裁定的重大污染广播；
- 严重违约与拒赔；
- 身份伪造与隐瞒控制权；
- 建筑故意荒废导致公共伤害；
- 重大治理欺诈。

---

## 四、成就模型映射

### 成就的现实结构建议
每项成就应至少包含：
- 唯一 ID；
- 成就类型；
- 触发条件；
- 授予证据；
- 授予方；
- 当前状态（有效 / 争议 / 褫夺）。

### 示例映射表

| 成就类型 | 现实基础 | 世界观称号示例 |
|---------|---------|---------------|
| 基础设施长期稳定运行 | uptime + 公共依赖度 | “永灯守望者” |
| 黑区重大探测发现 | 探测记录 + 多方确认 | “深渊寻星者” |
| 连续多年无严重违约 | 订单与仲裁历史 | “无瑕契约持守者” |
| 战时保护走廊 | 救援记录 + 广播见证 | “航道庇护者” |
| 共建联邦巨构 | 多方治理记录 + 建筑竣工记录 | “巨构缔造者” |
| 重大历史污点 | 裁决 + 审计证据 | “污染纪元始作俑者” |

## 五、负面记录与修复机制

### 负面记录的设计原则
- 不是所有失败都应成为污点；
- 必须区分不可抗灾害与主观责任；
- 必须提供修复路径；
- 但不应允许简单清零。

### 修复来源示例
- 主动赔付；
- 公共道歉；
- 长期公共服务弥补；
- 配合翻案与真相公开；
- 在更大危机中承担修复责任。

### 仍应保留的痕迹
即便修复成功，也应在历史记录中保留：
- “曾发生过什么”；
- “后来怎样被修复”；
- “是谁承担了代价”。

## 六、与其他系统的接口映射

### 对 02 身份系统
- 声誉归属必须绑定合法身份链；
- 继承时要区分“声誉展示权”与“责任延续范围”。

### 对 04 治理系统
- 勋章授予与褫夺需经治理程序或见证机制；
- 污点记录需有裁决依据与上诉入口。

### 对 07 经济系统
- 履约声誉影响报价、质押比例、保险费率；
- 污点记录影响市场准入与担保能力。

### 对 06 建筑系统
- 公共建筑维护历史可形成成就；
- 建筑废墟与修复过程是声誉证据来源。

## 七、设计理由总结

声誉与成就之所以必须从“积分系统”升级为“公共记忆系统”，是因为一个复杂宇宙里：
- 未来选择依赖过去证据；
- 共同体信任依赖长期历史；
- 荣耀若不可验证，就会被宣传绑架；
- 罪责若不可保留，就会被高资源主体不断洗白。

因此，本章本质上处理的是宇宙如何保存对自己的解释权。

| 术语 | 含义 | 对应现实 |
|------|------|---------|
| 履约声誉 | 服务与交易层面的可信度 | SLA / 成功率 / 赔付记录 |
| 公共声誉 | 对共同体贡献的历史评价 | 公共服务贡献日志 |
| 污点记录 | 重大负面行为留痕 | Incident + Ruling |
| 成就授予 | 正式认可的历史证明 | Award / Badge / Certification |
| 褫夺 | 撤销错误或失格荣誉 | Revocation / Reclassification |
